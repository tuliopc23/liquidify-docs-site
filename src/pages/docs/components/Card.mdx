# Card

Flexible container component for grouping related content. Provides visual boundaries and hierarchy with Apple's liquid glass aesthetics.

## Import

```tsx
import { Card } from "liquidify-react/card";
```

## Basic Usage

<CardBasicDemo client:load />

```tsx
import { Card } from "liquidify-react/card";

export default function Example() {
  return (
    <Card variant="glass" padded>
      <h3>Card Title</h3>
      <p>Card content with liquid glass styling.</p>
    </Card>
  );
}
```

## Variants

The Card component offers three visual variants:

- **solid**: Opaque background with subtle border
- **glass**: Translucent liquid glass with backdrop blur
- **elevated**: Raised appearance with shadow

<CardVariantsDemo client:load />

```tsx
<Card variant="solid" padded>Solid Card</Card>
<Card variant="glass" padded>Glass Card</Card>
<Card variant="elevated" padded>Elevated Card</Card>
```

## With Padding

The `padded` prop adds consistent internal spacing:



```tsx
<Card variant="glass">No padding</Card>
<Card variant="glass" padded>With padding</Card>
```

## Complex Content



```tsx
<Card variant="glass" padded>
  <h3>Featured Article</h3>
  <p className="text-muted">Published on September 30, 2024</p>
  <p>
    Complex content with headings, metadata, and actions
    within a liquid glass container.
  </p>
  <div style={{ display: "flex", gap: "0.5rem" }}>
    <button>Read More</button>
    <button>Share</button>
  </div>
</Card>
```

## API Reference

### Props

- `variant`: `"solid" | "glass" | "elevated"` - Visual style variant (default: `"solid"`)
- `padded`: `boolean` - Add internal padding (default: `false`)
- All standard HTML div attributes are supported

### Styling

The Card component uses a simple div-based structure. For complex layouts, compose your own heading/content structure using standard HTML elements:

```tsx
<Card variant="glass" padded>
  <header>
    <h2>Title</h2>
    <p>Subtitle</p>
  </header>
  <main>
    <p>Content</p>
  </main>
  <footer>
    <button>Action</button>
  </footer>
</Card>
```

### Accessibility

- Semantic HTML structure with proper heading hierarchy
- Sufficient color contrast for readability
- Glass variant maintains legibility with backdrop blur
