# ToggleGroup

A collection of toggle buttons that can be configured to allow single selection (like radio buttons) or multiple selection (like checkboxes). Perfect for filter controls, formatting options, or any scenario where users need to select from multiple related options.

## Installation

```bash
npm install liquidify-react
```

## Import

```tsx
import { ToggleGroup } from "liquidify-react/ark-ui/toggleGroup";
```

## Usage

### Basic Toggle Group (Multiple Selection)

```tsx
import { useState } from "react";
import { ToggleGroup } from "liquidify-react/ark-ui/toggleGroup";

export default function BasicToggleGroup() {
  const [values, setValues] = useState(["bold"]);

  return (
    <ToggleGroup.Root value={values} onValueChange={setValues}>
      <ToggleGroup.Item value="bold">Bold</ToggleGroup.Item>
      <ToggleGroup.Item value="italic">Italic</ToggleGroup.Item>
      <ToggleGroup.Item value="underline">Underline</ToggleGroup.Item>
    </ToggleGroup.Root>
  );
}
```

### Text Formatting Toggle Group

```tsx
import { useState } from "react";
import { ToggleGroup } from "liquidify-react/ark-ui/toggleGroup";

export default function TextFormattingToggleGroup() {
  const [formatting, setFormatting] = useState(["bold"]);

  return (
    <div>
      <ToggleGroup.Root value={formatting} onValueChange={setFormatting}>
        <ToggleGroup.Item value="bold">
          <strong>B</strong>
        </ToggleGroup.Item>
        <ToggleGroup.Item value="italic">
          <em>I</em>
        </ToggleGroup.Item>
        <ToggleGroup.Item value="underline">
          <u>U</u>
        </ToggleGroup.Item>
        <ToggleGroup.Item value="strikethrough">
          <s>S</s>
        </ToggleGroup.Item>
      </ToggleGroup.Root>

      <div style={{ marginTop: "1rem" }}>
        <p>Selected formatting: {formatting.join(", ") || "None"}</p>
        <p
          style={{
            fontWeight: formatting.includes("bold") ? "bold" : "normal",
            fontStyle: formatting.includes("italic") ? "italic" : "normal",
            textDecoration:
              [
                formatting.includes("underline") ? "underline" : "",
                formatting.includes("strikethrough") ? "line-through" : "",
              ]
                .filter(Boolean)
                .join(" ") || "none",
          }}
        >
          Sample text with applied formatting
        </p>
      </div>
    </div>
  );
}
```

### Filter Toggle Group

```tsx
import { useState } from "react";
import { ToggleGroup } from "liquidify-react/ark-ui/toggleGroup";

export default function FilterToggleGroup() {
  const [filters, setFilters] = useState(["active"]);

  const items = [
    { id: 1, name: "Item 1", status: "active", category: "work" },
    { id: 2, name: "Item 2", status: "completed", category: "personal" },
    { id: 3, name: "Item 3", status: "active", category: "work" },
    { id: 4, name: "Item 4", status: "pending", category: "personal" },
  ];

  const filteredItems = items.filter(
    (item) => filters.includes(item.status) || filters.includes(item.category),
  );

  return (
    <div>
      <div style={{ marginBottom: "1rem" }}>
        <h4>Status Filters:</h4>
        <ToggleGroup.Root value={filters} onValueChange={setFilters}>
          <ToggleGroup.Item value="active">Active</ToggleGroup.Item>
          <ToggleGroup.Item value="completed">Completed</ToggleGroup.Item>
          <ToggleGroup.Item value="pending">Pending</ToggleGroup.Item>
        </ToggleGroup.Root>
      </div>

      <div style={{ marginBottom: "1rem" }}>
        <h4>Category Filters:</h4>
        <ToggleGroup.Root value={filters} onValueChange={setFilters}>
          <ToggleGroup.Item value="work">Work</ToggleGroup.Item>
          <ToggleGroup.Item value="personal">Personal</ToggleGroup.Item>
        </ToggleGroup.Root>
      </div>

      <div>
        <h4>Results ({filteredItems.length}):</h4>
        {filteredItems.map((item) => (
          <div
            key={item.id}
            style={{ padding: "0.5rem", border: "1px solid #eee", margin: "0.25rem 0" }}
          >
            {item.name} - {item.status} ({item.category})
          </div>
        ))}
      </div>
    </div>
  );
}
```

## API Reference

### ToggleGroup.Root

The root container for the toggle group.

#### Props

| Prop            | Type                                  | Default        | Description                                   |
| --------------- | ------------------------------------- | -------------- | --------------------------------------------- |
| `type`          | `"single" \| "multiple"`              | `"multiple"`   | Whether to allow single or multiple selection |
| `value`         | `string \| string[]`                  | -              | The controlled value(s)                       |
| `defaultValue`  | `string \| string[]`                  | -              | The default value(s) when uncontrolled        |
| `onValueChange` | `(value: string \| string[]) => void` | -              | Callback fired when selection changes         |
| `disabled`      | `boolean`                             | `false`        | Whether the entire group is disabled          |
| `loop`          | `boolean`                             | `true`         | Whether keyboard navigation should loop       |
| `orientation`   | `"horizontal" \| "vertical"`          | `"horizontal"` | The orientation of the group                  |

### ToggleGroup.Item

An individual toggle item within the group.

#### Props

| Prop             | Type        | Default | Description                           |
| ---------------- | ----------- | ------- | ------------------------------------- |
| `value`          | `string`    | -       | The unique value for this toggle item |
| `disabled`       | `boolean`   | `false` | Whether this item is disabled         |
| `children`       | `ReactNode` | -       | The toggle item content               |
| All button props | -           | -       | Standard HTML button attributes       |

## Accessibility

The ToggleGroup component includes comprehensive accessibility features:

- **ARIA Roles**: Implements appropriate roles for group and toggle items
- **Keyboard Navigation**: Arrow keys for navigation, Space/Enter for selection
- **Focus Management**: Proper focus handling and visual indicators
- **Screen Reader Support**: Announces selection changes and group information
- **High Contrast**: Compatible with high contrast themes

### Keyboard Shortcuts

| Key                | Action                                     |
| ------------------ | ------------------------------------------ |
| `Tab`              | Move focus into/out of the toggle group    |
| `Arrow Left/Right` | Navigate between toggle items (horizontal) |
| `Arrow Up/Down`    | Navigate between toggle items (vertical)   |
| `Space` / `Enter`  | Toggle the focused item                    |
| `Home`             | Focus the first toggle item                |
| `End`              | Focus the last toggle item                 |

## Best Practices

1. **Clear Labels**: Use descriptive text or icons that clearly indicate each option
2. **Logical Grouping**: Group related options that make sense together
3. **Visual Consistency**: Maintain consistent styling across all toggle items
4. **Selection Type**: Choose single vs multiple selection based on use case
5. **State Indication**: Make selected states clearly visible and distinct
6. **Responsive Design**: Consider how toggle groups adapt to different screen sizes
7. **Performance**: For large lists, consider virtualization or pagination
