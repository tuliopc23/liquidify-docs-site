# Field

Form field wrapper that provides consistent labeling, validation, and help text. Essential building block for accessible forms.

## Import

```tsx
import { Field } from "liquidify-react/ark-ui/field";
```

## Basic Usage

```tsx
export default function Example() {
  return (
    <Field.Root>
      <Field.Label>Email Address</Field.Label>
      <Field.Input type="email" placeholder="Enter your email" />
      <Field.HelperText>We'll never share your email with anyone.</Field.HelperText>
    </Field.Root>
  );
}
```

## With Validation

```tsx
import { useState } from "react";

export default function ValidationExample() {
  const [email, setEmail] = useState("");
  const [error, setError] = useState("");

  const validateEmail = (value) => {
    if (!value) {
      setError("Email is required");
    } else if (!/\S+@\S+\.\S+/.test(value)) {
      setError("Please enter a valid email address");
    } else {
      setError("");
    }
  };

  return (
    <Field.Root invalid={!!error}>
      <Field.Label>Email Address</Field.Label>
      <Field.Input
        type="email"
        value={email}
        onChange={(e) => {
          setEmail(e.target.value);
          validateEmail(e.target.value);
        }}
        placeholder="Enter your email"
      />
      {error ? (
        <Field.ErrorText>{error}</Field.ErrorText>
      ) : (
        <Field.HelperText>Enter a valid email address</Field.HelperText>
      )}
    </Field.Root>
  );
}
```

## Required Field

```tsx
export default function RequiredExample() {
  return (
    <Field.Root required>
      <Field.Label>
        Full Name
        <Field.RequiredIndicator>*</Field.RequiredIndicator>
      </Field.Label>
      <Field.Input placeholder="Enter your full name" />
      <Field.HelperText>This field is required</Field.HelperText>
    </Field.Root>
  );
}
```

## API Reference

### Root Props

- `required`: `boolean` - Mark field as required
- `disabled`: `boolean` - Disable all field inputs
- `invalid`: `boolean` - Mark field as having validation errors
- `readOnly`: `boolean` - Make field read-only

### Accessibility

- Automatic label association with inputs
- ARIA attributes for validation states
- Screen reader support for help and error text
- Proper focus management and keyboard navigation
