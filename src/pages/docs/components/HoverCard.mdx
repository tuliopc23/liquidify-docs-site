# HoverCard

A floating card that appears when hovering over an element, perfect for contextual information and previews.

## Installation

```bash
bun add liquidify-react
```

## Import

```tsx
import { HoverCard } from "liquidify-react/hover-card"
```

## Basic Usage

Display contextual information when hovering over a trigger element:

```tsx
import { HoverCard } from "liquidify-react/hover-card"
import { Button } from "liquidify-react/button"

export function BasicExample() {
  return (
    <HoverCard.Root>
      <HoverCard.Trigger asChild>
        <Button variant="outline">Hover me</Button>
      </HoverCard.Trigger>
      <HoverCard.Positioner>
        <HoverCard.Content>
          <HoverCard.Arrow />
          <div style={{ padding: "12px" }}>
            <h4>Contextual Information</h4>
            <p>This card appears when you hover over the trigger element.</p>
          </div>
        </HoverCard.Content>
      </HoverCard.Positioner>
    </HoverCard.Root>
  )
}
```

## User Profile Preview

Create rich profile previews for user mentions or avatars:

```tsx
import { HoverCard } from "liquidify-react/hover-card"
import { Avatar } from "liquidify-react/avatar"

export function UserProfileExample() {
  return (
    <HoverCard.Root>
      <HoverCard.Trigger asChild>
        <span style={{ cursor: "pointer", color: "blue" }}>@johndoe</span>
      </HoverCard.Trigger>
      <HoverCard.Positioner>
        <HoverCard.Content>
          <HoverCard.Arrow />
          <div style={{ padding: "16px", display: "flex", gap: "12px" }}>
            <Avatar.Root>
              <Avatar.Image src="https://github.com/johndoe.png" alt="John Doe" />
              <Avatar.Fallback>JD</Avatar.Fallback>
            </Avatar.Root>
            <div>
              <h4 style={{ margin: "0 0 4px 0" }}>John Doe</h4>
              <p style={{ margin: "0 0 8px 0", color: "gray" }}>@johndoe</p>
              <p style={{ margin: 0, fontSize: "14px" }}>
                Software Engineer passionate about UI/UX and developer tools.
              </p>
            </div>
          </div>
        </HoverCard.Content>
      </HoverCard.Positioner>
    </HoverCard.Root>
  )
}
```

## Controlled Open State

Control the hover card's visibility programmatically:

```tsx
import { useState } from "react"
import { HoverCard } from "liquidify-react/hover-card"
import { Button } from "liquidify-react/button"

export function ControlledExample() {
  const [open, setOpen] = useState(false)

  return (
    <div>
      <HoverCard.Root open={open} onOpenChange={setOpen}>
        <HoverCard.Trigger asChild>
          <Button variant="outline">Controlled Hover Card</Button>
        </HoverCard.Trigger>
        <HoverCard.Positioner>
          <HoverCard.Content>
            <HoverCard.Arrow />
            <div style={{ padding: "12px" }}>
              <p>This hover card is controlled by external state.</p>
              <Button size="sm" onClick={() => setOpen(false)}>
                Close
              </Button>
            </div>
          </HoverCard.Content>
        </HoverCard.Positioner>
      </HoverCard.Root>
      
      <div style={{ marginTop: "16px" }}>
        <Button onClick={() => setOpen(!open)}>
          {open ? "Hide" : "Show"} Hover Card
        </Button>
      </div>
    </div>
  )
}
```

## Custom Positioning

Customize the hover card's position and behavior:

```tsx
import { HoverCard } from "liquidify-react/hover-card"
import { Button } from "liquidify-react/button"

export function PositioningExample() {
  return (
    <HoverCard.Root openDelay={200} closeDelay={100}>
      <HoverCard.Trigger asChild>
        <Button variant="outline">Custom Positioning</Button>
      </HoverCard.Trigger>
      <HoverCard.Positioner 
        side="top" 
        align="center"
        sideOffset={8}
        alignOffset={0}
      >
        <HoverCard.Content>
          <HoverCard.Arrow />
          <div style={{ padding: "12px" }}>
            <p>This hover card appears above the trigger with custom delays.</p>
          </div>
        </HoverCard.Content>
      </HoverCard.Positioner>
    </HoverCard.Root>
  )
}
```

## API Reference

### HoverCard.Root

The root container for the hover card.

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `open` | `boolean` | `false` | Whether the hover card is open |
| `onOpenChange` | `(open: boolean) => void` | - | Callback when open state changes |
| `openDelay` | `number` | `700` | Delay before opening (ms) |
| `closeDelay` | `number` | `300` | Delay before closing (ms) |

### HoverCard.Trigger

The element that triggers the hover card. Must be used with `asChild`.

### HoverCard.Positioner

Controls the positioning of the hover card content.

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `side` | `"top" \| "right" \| "bottom" \| "left"` | `"bottom"` | Preferred side to render |
| `align` | `"start" \| "center" \| "end"` | `"center"` | Alignment relative to trigger |
| `sideOffset` | `number` | `4` | Distance from trigger element |
| `alignOffset` | `number` | `0` | Additional alignment offset |

### HoverCard.Content

The main content area of the hover card.

### HoverCard.Arrow

An optional arrow pointing to the trigger element.

## Accessibility

The HoverCard component follows WAI-ARIA guidelines:

- Uses `role="tooltip"` for the content
- Properly links trigger and content with `aria-describedby`
- Supports keyboard navigation and focus management
- Respects `prefers-reduced-motion` for animations
- Provides appropriate delays for hover interactions

## Best Practices

- Use for supplementary information that doesn't require interaction
- Keep content concise and scannable
- Provide sufficient hover delay to prevent accidental triggers
- Consider mobile users who can't hover - provide alternative access methods
- Ensure hover cards don't block important interface elements
