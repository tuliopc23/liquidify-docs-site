# Tour

An interactive guided tour component that helps users discover features and navigate through your application. Perfect for onboarding new users, introducing new features, or providing contextual help with step-by-step guidance.

## Installation

```bash
npm install @liquidify/components
```

## Import

```tsx
import { Tour } from "@liquidify/components/tour";
```

## Usage

### Basic Tour

```tsx
import { useState } from "react";
import { Tour } from "@liquidify/components/tour";

export default function BasicTour() {
  const [isOpen, setIsOpen] = useState(false);
  const [currentStep, setCurrentStep] = useState(0);

  const steps = [
    {
      target: "#step1",
      title: "Welcome!",
      description: "This is your dashboard where you can see all your projects."
    },
    {
      target: "#step2", 
      title: "Create New",
      description: "Click here to create a new project or item."
    },
    {
      target: "#step3",
      title: "Settings",
      description: "Access your account settings and preferences here."
    }
  ];

  return (
    <div>
      <button onClick={() => setIsOpen(true)}>
        Start Tour
      </button>
      
      <div style={{ padding: "2rem", display: "flex", gap: "2rem" }}>
        <div id="step1" style={{ padding: "1rem", border: "1px solid #ccc" }}>
          Dashboard Content
        </div>
        <div id="step2" style={{ padding: "1rem", border: "1px solid #ccc" }}>
          Create Button
        </div>
        <div id="step3" style={{ padding: "1rem", border: "1px solid #ccc" }}>
          Settings Menu
        </div>
      </div>

      <Tour.Root 
        open={isOpen} 
        onOpenChange={setIsOpen}
        step={currentStep}
        onStepChange={setCurrentStep}
      >
        <Tour.Spotlight />
        <Tour.Positioner>
          <Tour.Content>
            <Tour.Arrow>
              <Tour.ArrowTip />
            </Tour.Arrow>
            
            <Tour.Title>{steps[currentStep]?.title}</Tour.Title>
            <Tour.Description>{steps[currentStep]?.description}</Tour.Description>
            
            <div style={{ display: "flex", justifyContent: "space-between", marginTop: "1rem" }}>
              <button 
                onClick={() => setCurrentStep(Math.max(0, currentStep - 1))}
                disabled={currentStep === 0}
              >
                Previous
              </button>
              
              <span>{currentStep + 1} of {steps.length}</span>
              
              {currentStep < steps.length - 1 ? (
                <button onClick={() => setCurrentStep(currentStep + 1)}>
                  Next
                </button>
              ) : (
                <button onClick={() => setIsOpen(false)}>
                  Finish
                </button>
              )}
            </div>
            
            <Tour.CloseTrigger>√ó</Tour.CloseTrigger>
          </Tour.Content>
        </Tour.Positioner>
      </Tour.Root>
    </div>
  );
}
```

### Conditional Tour Steps

```tsx
import { useState } from "react";

  const [tourOpen, setTourOpen] = useState(false);

      { target: "#tutorial", title: "Start Here", description: "Check out our beginner tutorial." },

      ...commonSteps,

  return (
```

## API Reference

### Tour.Root

The root container for the tour component.

#### Props

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `open` | `boolean` | - | Whether the tour is visible |
| `onOpenChange` | `(open: boolean) => void` | - | Callback fired when visibility changes |
| `step` | `number` | `0` | Current step index |
| `onStepChange` | `(step: number) => void` | - | Callback fired when step changes |
| `steps` | `TourStep[]` | - | Array of tour step configurations |

### Tour.Positioner

The positioning wrapper for tour content.

#### Props

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `children` | `ReactNode` | - | The positioner content |

### Tour.Content

The main content container for the tour step.

#### Props

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `children` | `ReactNode` | - | The tour content |

### Tour.Title

The title of the current tour step.

#### Props

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `children` | `ReactNode` | - | The title content |

### Tour.Description

The description text for the current tour step.

#### Props

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `children` | `ReactNode` | - | The description content |

### Tour.ActionTrigger

A button for tour actions (next, previous, finish).

#### Props

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `onClick` | `() => void` | - | Action click handler |
| `children` | `ReactNode` | - | The action trigger content |

### Tour.CloseTrigger

A button to close/exit the tour.

#### Props

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `children` | `ReactNode` | - | The close trigger content |

### Tour.Spotlight

A visual spotlight effect that highlights the target element.

#### Props

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| All standard div props | - | - | Standard HTML div attributes |

### Tour.Arrow

The pointing arrow for the tour content.

#### Props

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `children` | `ReactNode` | - | The arrow content (usually ArrowTip) |

### Tour.ArrowTip

The tip/point of the tour arrow.

#### Props

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| All standard div props | - | - | Standard HTML div attributes |

## Accessibility

The Tour component includes comprehensive accessibility features:

- **ARIA Labels**: Implements proper ARIA roles and labels for tour elements
- **Keyboard Navigation**: Full keyboard support for navigating through steps
- **Focus Management**: Manages focus appropriately during tour progression
- **Screen Reader Support**: Announces tour content and step changes
- **High Contrast**: Compatible with high contrast themes

### Keyboard Shortcuts

| Key | Action |
|-----|--------|
| `Escape` | Close the tour |
| `Tab` | Navigate between tour controls |
| `Enter` / `Space` | Activate focused button |
| `Arrow Left` | Previous step (when applicable) |
| `Arrow Right` | Next step (when applicable) |

## Best Practices

1. **Clear Progression**: Make it obvious how many steps remain and current progress
2. **Concise Content**: Keep step titles and descriptions brief and focused
3. **Visual Hierarchy**: Use proper heading levels and clear visual structure
4. **Skip Option**: Always provide a way to skip or exit the tour
5. **Responsive Design**: Ensure tours work well on all screen sizes
6. **Smart Positioning**: Position tour content to avoid covering important elements
7. **Progressive Disclosure**: Introduce features gradually, not all at once
8. **Contextual Timing**: Show tours at the right moment in the user journey
