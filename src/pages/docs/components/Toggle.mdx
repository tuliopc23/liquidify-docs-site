# Toggle

A two-state button that allows users to switch between an on and off state. Perfect for enabling/disabling features, settings, or any binary choice where the current state needs to be clearly visible.

## Installation

```bash
npm install liquidify-react
```

## Import

```tsx
import { Toggle } from "liquidify-react/ark-ui/toggle";
```

## Usage

### Basic Toggle

```tsx
import { Toggle } from "liquidify-react/ark-ui/toggle";

export default function BasicToggle() {
  return <Toggle.Root>Toggle me</Toggle.Root>;
}
```

### Controlled Toggle

```tsx
import { useState } from "react";
import { Toggle } from "liquidify-react/ark-ui/toggle";

export default function ControlledToggle() {
  const [isPressed, setIsPressed] = useState(false);

  return (
    <div>
      <Toggle.Root pressed={isPressed} onPressedChange={setIsPressed}>
        {isPressed ? "On" : "Off"}
      </Toggle.Root>
      <p>Current state: {isPressed ? "Pressed" : "Not pressed"}</p>
    </div>
  );
}
```

      <div style={{ display: "flex", justifyContent: "space-between", alignItems: "center" }}>

## API Reference

### Toggle.Root

The toggle button component.

#### Props

| Prop              | Type                         | Default | Description                                            |
| ----------------- | ---------------------------- | ------- | ------------------------------------------------------ |
| `pressed`         | `boolean`                    | -       | Whether the toggle is pressed (controlled)             |
| `defaultPressed`  | `boolean`                    | `false` | Whether the toggle is initially pressed (uncontrolled) |
| `onPressedChange` | `(pressed: boolean) => void` | -       | Callback fired when pressed state changes              |
| `disabled`        | `boolean`                    | `false` | Whether the toggle is disabled                         |
| `children`        | `ReactNode`                  | -       | The toggle content                                     |
| All button props  | -                            | -       | Standard HTML button attributes                        |

## Accessibility

The Toggle component includes comprehensive accessibility features:

- **ARIA States**: Implements `aria-pressed` to indicate toggle state
- **Keyboard Support**: Space and Enter keys toggle the state
- **Screen Reader Support**: Announces state changes to assistive technology
- **Focus Management**: Proper focus indicators and keyboard navigation
- **High Contrast**: Compatible with high contrast themes

### Keyboard Shortcuts

| Key           | Action                         |
| ------------- | ------------------------------ |
| `Space`       | Toggle the pressed state       |
| `Enter`       | Toggle the pressed state       |
| `Tab`         | Move focus to next element     |
| `Shift + Tab` | Move focus to previous element |

## Best Practices

1. **Clear Labels**: Use descriptive text or icons that clearly indicate the toggle's purpose
2. **Visual State**: Make the on/off states visually distinct and obvious
3. **Immediate Feedback**: Provide instant visual feedback when the state changes
4. **Consistent Behavior**: Use toggles consistently throughout your application
5. **Appropriate Use**: Use toggles for binary states, not for actions that require confirmation
6. **Accessibility**: Always provide appropriate ARIA labels for icon-only toggles
7. **Grouping**: Consider using ToggleGroup for related toggle options
