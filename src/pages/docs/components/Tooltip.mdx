# Tooltip

A floating text box that appears when users hover over, focus on, or touch an element. Tooltips provide supplementary information without cluttering the interface, perfect for explaining functionality or showing additional context.

## Installation

```bash
npm install liquidify-react
```

## Import

```tsx
import { Tooltip } from "liquidify-react/ark-ui/tooltip";
```

## Usage

### Basic Tooltip

<TooltipDemo client:only="react" />

```tsx
import { Tooltip } from "liquidify-react/ark-ui/tooltip";

export default function BasicTooltip() {
  return (
    <Tooltip.Root>
      <Tooltip.Trigger>
        <button>Hover me</button>
      </Tooltip.Trigger>
      <Tooltip.Positioner>
        <Tooltip.Content>
          This is a helpful tooltip message
        </Tooltip.Content>
      </Tooltip.Positioner>
    </Tooltip.Root>
  );
}
```

### Controlled Tooltip

```tsx
import { useState } from "react";
import { Tooltip } from "liquidify-react/ark-ui/tooltip";

export default function ControlledTooltip() {
  const [isOpen, setIsOpen] = useState(false);

  return (
    <div>
      <Tooltip.Root open={isOpen} onOpenChange={setIsOpen}>
        <Tooltip.Trigger>
          <button>Controlled tooltip</button>
        </Tooltip.Trigger>
        <Tooltip.Positioner>
          <Tooltip.Content>
            This tooltip is controlled by state
          </Tooltip.Content>
        </Tooltip.Positioner>
      </Tooltip.Root>
      
      <button 
        onClick={() => setIsOpen(!isOpen)}
        style={{ marginLeft: "1rem" }}
      >
        Toggle Tooltip
      </button>
    </div>
  );
}
```

### Positioned Tooltips

```tsx
import { Tooltip } from "liquidify-react/ark-ui/tooltip";

export default function PositionedTooltips() {
  const positions = [
    { placement: "top", label: "Top" },
    { placement: "bottom", label: "Bottom" },
    { placement: "left", label: "Left" },
    { placement: "right", label: "Right" },
    { placement: "top-start", label: "Top Start" },
    { placement: "top-end", label: "Top End" },
    { placement: "bottom-start", label: "Bottom Start" },
    { placement: "bottom-end", label: "Bottom End" },
  ];

  return (
    <div style={{ 
      display: "grid", 
      gridTemplateColumns: "repeat(4, 1fr)", 
      gap: "1rem",
      padding: "2rem"
    }}>
      {positions.map(({ placement, label }) => (
        <Tooltip.Root key={placement} positioning={{ placement }}>
          <Tooltip.Trigger>
            <button>{label}</button>
          </Tooltip.Trigger>
          <Tooltip.Positioner>
            <Tooltip.Content>
              Tooltip positioned {placement}
            </Tooltip.Content>
          </Tooltip.Positioner>
        </Tooltip.Root>
      ))}
    </div>
  );
}
```

## API Reference

### Tooltip.Root

The root container for the tooltip component.

#### Props

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `open` | `boolean` | - | Whether the tooltip is visible (controlled) |
| `defaultOpen` | `boolean` | `false` | Whether the tooltip is initially visible (uncontrolled) |
| `onOpenChange` | `(open: boolean) => void` | - | Callback fired when visibility changes |
| `openDelay` | `number` | `700` | Delay in ms before showing tooltip |
| `closeDelay` | `number` | `300` | Delay in ms before hiding tooltip |
| `disabled` | `boolean` | `false` | Whether the tooltip is disabled |
| `positioning` | `PositioningOptions` | - | Positioning configuration |

### Tooltip.Trigger

The element that triggers the tooltip on hover/focus.

#### Props

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `asChild` | `boolean` | `false` | Render as child element instead of button |
| `children` | `ReactNode` | - | The trigger content |

### Tooltip.Positioner

The positioning wrapper for the tooltip content.

#### Props

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `children` | `ReactNode` | - | The positioner content |

### Tooltip.Content

The actual tooltip content container.

#### Props

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `children` | `ReactNode` | - | The tooltip content |

## Positioning

The tooltip supports flexible positioning options:

```tsx
// Basic positioning
<Tooltip.Root positioning={{ placement: "top" }}>

// Advanced positioning
<Tooltip.Root positioning={{ 
  placement: "bottom-start",
  offset: { mainAxis: 8, crossAxis: 4 },
  gutter: 4
}}>
```

### Placement Options

- `top`, `bottom`, `left`, `right`
- `top-start`, `top-end`
- `bottom-start`, `bottom-end`
- `left-start`, `left-end`
- `right-start`, `right-end`

## Accessibility

The Tooltip component includes comprehensive accessibility features:

- **ARIA Labels**: Implements proper `aria-describedby` relationships
- **Keyboard Support**: Shows on focus, hides on escape
- **Screen Reader Support**: Content is announced when tooltip appears
- **High Contrast**: Compatible with high contrast themes
- **Touch Support**: Optimized for touch devices

### Keyboard Shortcuts

| Key | Action |
|-----|--------|
| `Tab` | Show tooltip when trigger receives focus |
| `Escape` | Hide tooltip |
| `Enter` / `Space` | Activate trigger (if it's a button) |

## Best Practices

1. **Concise Content**: Keep tooltip text brief and informative
2. **Essential Information**: Don't rely on tooltips for critical information
3. **Appropriate Triggers**: Use tooltips for helpful, not essential, information
4. **Performance**: Avoid complex content that might cause layout shifts
5. **Touch Devices**: Consider alternative patterns for mobile/touch devices
6. **Positioning**: Ensure tooltips don't cover important content
7. **Timing**: Use appropriate delays to avoid flickering on quick mouse movements
