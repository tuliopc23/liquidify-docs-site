# Button

Interactive elements that trigger actions when clicked. Essential for forms, navigation, and user interactions with Apple HIG-compliant styling.

## Import

```tsx
import { Button } from "liquidify-react/button";
```

## Basic Usage

<ButtonBasicDemo client:load />

```tsx
import { Button } from "liquidify-react/button";

export default function Example() {
  return (
    <div style={{ display: "flex", gap: "1rem" }}>
      <Button variant="filled" tone="accent">Filled Accent</Button>
      <Button variant="tinted" tone="neutral">Tinted Neutral</Button>
      <Button variant="plain" tone="neutral">Plain</Button>
    </div>
  );
}
```

## Variants and Tones

The Button uses Apple's HIG-compliant API with `variant` (visual emphasis) and `tone` (semantic meaning).

### Variants

- **filled**: High emphasis, solid background
- **tinted**: Medium emphasis, tinted background
- **plain**: Low emphasis, transparent background

### Tones

- **accent**: Primary actions (blue accent)
- **neutral**: Secondary actions (gray)
- **destructive**: Destructive actions (red)

## Legacy API (Deprecated)

For backward compatibility, the old API still works but shows deprecation warnings:

```tsx
// ⚠️ DEPRECATED - Will be removed in future versions
<Button variant="primary">Primary</Button>     // → variant="filled" tone="accent"
<Button variant="secondary">Secondary</Button> // → variant="tinted" tone="neutral"
<Button variant="ghost">Ghost</Button>         // → variant="plain" tone="neutral"
<Button variant="danger">Danger</Button>       // → variant="filled" tone="destructive"
```

## API Reference

### Props

- `variant`: `"filled" | "tinted" | "plain"` - Visual emphasis level
- `tone`: `"accent" | "neutral" | "destructive"` - Semantic color tone
- `size`: `"compact" | "regular" | "large"` - Button size (also accepts legacy `"sm" | "md" | "lg" | "xl"`)
- `as`: `React.ElementType` - Polymorphic element type (e.g., `"a"` for links)
- `icon`: `React.ReactNode` - Icon element to render
- `iconPosition`: `"start" | "end"` - Icon placement
- `loading`: `boolean` - Show loading state
- `disabled`: `boolean` - Disable button interaction
- `magneticHover`: `boolean` - Enable magnetic hover effect
- `magneticStrength`: `number` - Magnetic effect strength
- Standard HTML button attributes are supported

### Accessibility

- Minimum 44×44px touch target (enforced via size variants)
- Full keyboard navigation with Enter and Space
- Disabled state with proper `aria-disabled`
- Loading state with `aria-busy`
- Focus ring using accent color
- Icon-only buttons require `aria-label`
