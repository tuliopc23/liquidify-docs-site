# Menu

A dropdown menu component with support for nested submenus and contextual actions.

## Installation

```bash
bun add @liquidify/components
```

## Import

```tsx
import { Menu } from "@liquidify/components/menu"
```

## Basic Usage

Create a simple dropdown menu with actions:

```tsx
import { Menu } from "@liquidify/components/menu"
import { Button } from "@liquidify/components/button"

export function BasicExample() {
  return (
    <Menu.Root>
      <Menu.Trigger asChild>
        <Button variant="outline">Options</Button>
      </Menu.Trigger>
      <Menu.Positioner>
        <Menu.Content>
          <Menu.Item onClick={() => console.log("Profile")}>
            Profile
          </Menu.Item>
          <Menu.Item onClick={() => console.log("Settings")}>
            Settings
          </Menu.Item>
          <Menu.Separator />
          <Menu.Item onClick={() => console.log("Logout")}>
            Logout
          </Menu.Item>
        </Menu.Content>
      </Menu.Positioner>
    </Menu.Root>
  )
}
```

## Context Menu

Create a right-click context menu:

```tsx
import { Menu } from "@liquidify/components/menu"

export function ContextMenuExample() {
  return (
    <Menu.Root>
      <Menu.ContextTrigger asChild>
        <div 
          style={{ 
            padding: "40px", 
            border: "2px dashed #ccc", 
            textAlign: "center" 
          }}
        >
          Right-click me for context menu
        </div>
      </Menu.ContextTrigger>
      <Menu.Positioner>
        <Menu.Content>
          <Menu.Item>Copy</Menu.Item>
          <Menu.Item>Cut</Menu.Item>
          <Menu.Item>Paste</Menu.Item>
          <Menu.Separator />
          <Menu.Item>Delete</Menu.Item>
        </Menu.Content>
      </Menu.Positioner>
    </Menu.Root>
  )
}
```

## Nested Submenus

Create menus with nested submenus:

```tsx
import { Menu } from "@liquidify/components/menu"
import { Button } from "@liquidify/components/button"

export function NestedMenuExample() {
  return (
    <Menu.Root>
      <Menu.Trigger asChild>
        <Button variant="outline">File</Button>
      </Menu.Trigger>
      <Menu.Positioner>
        <Menu.Content>
          <Menu.Item>New File</Menu.Item>
          <Menu.Item>Open File</Menu.Item>
          
          <Menu.Root>
            <Menu.TriggerItem>Export</Menu.TriggerItem>
            <Menu.Positioner>
              <Menu.Content>
                <Menu.Item>Export as PDF</Menu.Item>
                <Menu.Item>Export as Image</Menu.Item>
                <Menu.Item>Export as JSON</Menu.Item>
              </Menu.Content>
            </Menu.Positioner>
          </Menu.Root>
          
          <Menu.Separator />
          <Menu.Item>Save</Menu.Item>
          <Menu.Item>Save As...</Menu.Item>
        </Menu.Content>
      </Menu.Positioner>
    </Menu.Root>
  )
}
```

## Controlled Menu

Control the menu's open state programmatically:

```tsx
import { useState } from "react"
import { Menu } from "@liquidify/components/menu"
import { Button } from "@liquidify/components/button"

export function ControlledMenuExample() {
  const [open, setOpen] = useState(false)

  return (
    <div>
      <Menu.Root open={open} onOpenChange={setOpen}>
        <Menu.Trigger asChild>
          <Button variant="outline">Controlled Menu</Button>
        </Menu.Trigger>
        <Menu.Positioner>
          <Menu.Content>
            <Menu.Item onClick={() => setOpen(false)}>
              Close Menu
            </Menu.Item>
            <Menu.Item>Keep Open</Menu.Item>
          </Menu.Content>
        </Menu.Positioner>
      </Menu.Root>
      
      <div style={{ marginTop: "16px" }}>
        <Button onClick={() => setOpen(!open)}>
          {open ? "Close" : "Open"} Menu
        </Button>
      </div>
    </div>
  )
}
```

## API Reference

### Menu.Root

The root container for the menu.

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `open` | `boolean` | `false` | Whether the menu is open |
| `onOpenChange` | `(open: boolean) => void` | - | Callback when open state changes |
| `closeOnSelect` | `boolean` | `true` | Whether to close menu when item is selected |

### Menu.Trigger

The button that triggers the menu dropdown.

### Menu.ContextTrigger

The element that triggers the menu on right-click.

### Menu.Positioner

Controls the positioning of the menu content.

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `side` | `"top" \| "right" \| "bottom" \| "left"` | `"bottom"` | Preferred side to render |
| `align` | `"start" \| "center" \| "end"` | `"start"` | Alignment relative to trigger |

### Menu.Content

The container for menu items.

### Menu.Item

An individual menu item that can be clicked.

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `onClick` | `() => void` | - | Click handler |
| `disabled` | `boolean` | `false` | Whether the item is disabled |

### Menu.TriggerItem

A menu item that opens a submenu.

### Menu.Separator

A visual separator between menu items.

## Accessibility

The Menu component follows WAI-ARIA guidelines:

- Uses proper `role="menu"` and `role="menuitem"` attributes
- Supports keyboard navigation with arrow keys
- Implements focus management and escape handling
- Provides screen reader announcements
- Supports both click and keyboard activation

## Best Practices

- Keep menu items concise and clearly labeled
- Group related actions together with separators
- Use consistent iconography and shortcuts
- Limit nesting depth to maintain usability
- Consider mobile touch targets for menu items
- Provide keyboard shortcuts for frequently used actions
