# Tabs

A collection of layered sections of content—known as tab panels—that are displayed one at a time. Tabs organize content into multiple sections allowing users to navigate between different views while preserving the context of their current location.

## Installation

```bash
npm install @liquidify/components
```

## Import

```tsx
import { Tabs } from "@liquidify/components/tabs";
```

## Usage

### Basic Tabs

```tsx
import { Tabs } from "@liquidify/components/tabs";

export default function BasicTabs() {
  return (
    <Tabs.Root defaultValue="overview">
      <Tabs.List>
        <Tabs.Trigger value="overview">Overview</Tabs.Trigger>
        <Tabs.Trigger value="details">Details</Tabs.Trigger>
        <Tabs.Trigger value="settings">Settings</Tabs.Trigger>
      </Tabs.List>
      <Tabs.Content value="overview">
        <h3>Project Overview</h3>
        <p>A comprehensive view of your project's current status and key metrics.</p>
      </Tabs.Content>
      <Tabs.Content value="details">
        <h3>Project Details</h3>
        <p>Detailed information about project specifications and requirements.</p>
      </Tabs.Content>
      <Tabs.Content value="settings">
        <h3>Project Settings</h3>
        <p>Configure project preferences and advanced options.</p>
      </Tabs.Content>
    </Tabs.Root>
  );
}
```

### Controlled Tabs

```tsx
import { useState } from "react";
import { Tabs } from "@liquidify/components/tabs";

export default function ControlledTabs() {
  const [activeTab, setActiveTab] = useState("dashboard");

  return (
    <Tabs.Root value={activeTab} onValueChange={setActiveTab}>
      <Tabs.List>
        <Tabs.Trigger value="dashboard">Dashboard</Tabs.Trigger>
        <Tabs.Trigger value="analytics">Analytics</Tabs.Trigger>
        <Tabs.Trigger value="reports">Reports</Tabs.Trigger>
      </Tabs.List>
      <Tabs.Content value="dashboard">
        <div>
          <h2>Dashboard</h2>
          <p>Welcome to your main dashboard with key insights.</p>
        </div>
      </Tabs.Content>
      <Tabs.Content value="analytics">
        <div>
          <h2>Analytics</h2>
          <p>Detailed analytics and performance metrics.</p>
        </div>
      </Tabs.Content>
      <Tabs.Content value="reports">
        <div>
          <h2>Reports</h2>
          <p>Generate and view comprehensive reports.</p>
        </div>
      </Tabs.Content>
    </Tabs.Root>
  );
}
```

### Lazy Loading Tabs

```tsx
import { useState } from "react";
import { Tabs } from "@liquidify/components/tabs";

const ExpensiveComponent = ({ title }: { title: string }) => {
  console.log(`Loading ${title} content...`);
  return (
    <div>
      <h3>{title}</h3>
      <p>This content is loaded only when the tab becomes active.</p>
    </div>
  );
};

export default function LazyLoadingTabs() {
  const [activeTab, setActiveTab] = useState("tab1");

  return (
    <Tabs.Root value={activeTab} onValueChange={setActiveTab}>
      <Tabs.List>
        <Tabs.Trigger value="tab1">Quick Load</Tabs.Trigger>
        <Tabs.Trigger value="tab2">Heavy Content</Tabs.Trigger>
        <Tabs.Trigger value="tab3">Data Visualization</Tabs.Trigger>
      </Tabs.List>
      <Tabs.Content value="tab1">
        <ExpensiveComponent title="Quick Content" />
      </Tabs.Content>
      <Tabs.Content value="tab2">
        {activeTab === "tab2" && <ExpensiveComponent title="Heavy Content" />}
      </Tabs.Content>
      <Tabs.Content value="tab3">
        {activeTab === "tab3" && <ExpensiveComponent title="Data Visualization" />}
      </Tabs.Content>
    </Tabs.Root>
  );
}
```

## API Reference

### Tabs.Root

The root container for the tabs component.

#### Props

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `defaultValue` | `string` | - | The initial tab value when uncontrolled |
| `value` | `string` | - | The controlled tab value |
| `onValueChange` | `(value: string) => void` | - | Callback fired when the tab value changes |
| `orientation` | `"horizontal" \| "vertical"` | `"horizontal"` | The orientation of the tabs |
| `disabled` | `boolean` | `false` | Whether the entire tabs component is disabled |
| `activationMode` | `"automatic" \| "manual"` | `"automatic"` | How tabs are activated |

### Tabs.List

The container for tab triggers.

#### Props

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `children` | `ReactNode` | - | The tab triggers |

### Tabs.Trigger

A button that activates a tab panel.

#### Props

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `value` | `string` | - | The unique value of the tab |
| `disabled` | `boolean` | `false` | Whether the tab trigger is disabled |
| `children` | `ReactNode` | - | The tab trigger content |

### Tabs.Content

The content area for a tab panel.

#### Props

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `value` | `string` | - | The value that corresponds to the tab trigger |
| `children` | `ReactNode` | - | The tab content |

### Tabs.Indicator

A visual indicator that shows the active tab.

#### Props

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| All standard HTML attributes | - | - | Standard HTML attributes |

## Accessibility

The Tabs component includes comprehensive accessibility features:

- **Keyboard Navigation**: Use arrow keys to navigate between tabs, Enter/Space to activate
- **Focus Management**: Proper focus handling with visible focus indicators
- **ARIA Labels**: Implements `tablist`, `tab`, and `tabpanel` roles
- **Screen Reader Support**: Announces tab changes and content relationships
- **Disabled State**: Properly handles disabled tabs with appropriate ARIA attributes

### Keyboard Shortcuts

| Key | Action |
|-----|--------|
| `Tab` | Move focus to the next focusable element |
| `Shift + Tab` | Move focus to the previous focusable element |
| `Arrow Left/Right` | Navigate between tabs (horizontal orientation) |
| `Arrow Up/Down` | Navigate between tabs (vertical orientation) |
| `Enter` / `Space` | Activate the focused tab |
| `Home` | Focus the first tab |
| `End` | Focus the last tab |

## Best Practices

1. **Clear Labels**: Use descriptive and concise labels for tabs
2. **Logical Order**: Arrange tabs in a logical sequence based on user workflow
3. **Consistent Content**: Ensure tab content is structured consistently
4. **Performance**: Implement lazy loading for heavy content in tabs
5. **Responsive Design**: Consider how tabs will behave on smaller screens
6. **State Management**: Use controlled tabs when you need to sync with external state
7. **Accessibility**: Always provide proper ARIA labels and keyboard navigation
