# Popover

A floating panel that displays rich content in a portal, triggered by user interaction.

## Installation

```bash
bun add liquidify-react
```

## Import

```tsx
import { Popover } from "liquidify-react/ark-ui/popover"
```

## Basic Usage

Create a simple popover with content:

<PopoverDemo client:only="react" />

```tsx
import { Popover } from "liquidify-react/ark-ui/popover"
import { Button } from "liquidify-react/button"

export function BasicExample() {
  return (
    <Popover.Root>
      <Popover.Trigger asChild>
        <Button variant="outline">Open Popover</Button>
      </Popover.Trigger>
      <Popover.Positioner>
        <Popover.Content>
          <Popover.Arrow />
          <div style={{ padding: "16px" }}>
            <h3>Popover Title</h3>
            <p>This is the popover content that appears when triggered.</p>
            <Popover.CloseTrigger asChild>
              <Button size="sm">Close</Button>
            </Popover.CloseTrigger>
          </div>
        </Popover.Content>
      </Popover.Positioner>
    </Popover.Root>
  )
}
```

## Form in Popover

Use popovers for inline forms and actions:

```tsx
import { useState } from "react"
import { Popover } from "liquidify-react/ark-ui/popover"
import { Button } from "liquidify-react/button"

export function FormExample() {
  const [name, setName] = useState("")
  const [email, setEmail] = useState("")

  const handleSubmit = (e) => {
    e.preventDefault()
    console.log("Submitted:", { name, email })
    // Close popover after submission
  }

  return (
    <Popover.Root>
      <Popover.Trigger asChild>
        <Button variant="outline">Add Contact</Button>
      </Popover.Trigger>
      <Popover.Positioner>
        <Popover.Content style={{ padding: "20px", width: "300px" }}>
          <Popover.Arrow />
          <h3 style={{ margin: "0 0 16px 0" }}>Add New Contact</h3>
          <form onSubmit={handleSubmit}>
            <div style={{ marginBottom: "12px" }}>
              <label style={{ display: "block", marginBottom: "4px" }}>Name</label>
              <input
                type="text"
                value={name}
                onChange={(e) => setName(e.target.value)}
                style={{ width: "100%", padding: "8px" }}
                required
              />
            </div>
            <div style={{ marginBottom: "16px" }}>
              <label style={{ display: "block", marginBottom: "4px" }}>Email</label>
              <input
                type="email"
                value={email}
                onChange={(e) => setEmail(e.target.value)}
                style={{ width: "100%", padding: "8px" }}
                required
              />
            </div>
            <div style={{ display: "flex", gap: "8px", justifyContent: "flex-end" }}>
              <Popover.CloseTrigger asChild>
                <Button variant="outline" size="sm">Cancel</Button>
              </Popover.CloseTrigger>
              <Button type="submit" size="sm">Add Contact</Button>
            </div>
          </form>
        </Popover.Content>
      </Popover.Positioner>
    </Popover.Root>
  )
}
```

## Controlled Popover

Control the popover's open state programmatically:

```tsx
import { useState } from "react"
import { Popover } from "liquidify-react/popover"
import { Button } from "liquidify-react/button"

export function ControlledExample() {
  const [open, setOpen] = useState(false)

  return (
    <div>
      <Popover.Root open={open} onOpenChange={setOpen}>
        <Popover.Trigger asChild>
          <Button variant="outline">Controlled Popover</Button>
        </Popover.Trigger>
        <Popover.Positioner>
          <Popover.Content>
            <Popover.Arrow />
            <div style={{ padding: "16px" }}>
              <p>This popover is controlled by external state.</p>
              <Button size="sm" onClick={() => setOpen(false)}>
                Close from Inside
              </Button>
            </div>
          </Popover.Content>
        </Popover.Positioner>
      </Popover.Root>
      
      <div style={{ marginTop: "16px" }}>
        <Button onClick={() => setOpen(!open)}>
          {open ? "Close" : "Open"} Popover Externally
        </Button>
      </div>
    </div>
  )
}
```

## Settings Panel

Create a settings panel using popover:

```tsx
import { useState } from "react"
import { Popover } from "liquidify-react/popover"
import { Button } from "liquidify-react/button"

export function SettingsExample() {
  const [notifications, setNotifications] = useState(true)
  const [darkMode, setDarkMode] = useState(false)
  const [autoSave, setAutoSave] = useState(true)

  return (
    <Popover.Root>
      <Popover.Trigger asChild>
        <Button variant="outline">⚙️ Settings</Button>
      </Popover.Trigger>
      <Popover.Positioner>
        <Popover.Content style={{ padding: "20px", width: "280px" }}>
          <Popover.Arrow />
          <h3 style={{ margin: "0 0 16px 0" }}>Settings</h3>
          
          <div style={{ display: "flex", flexDirection: "column", gap: "12px" }}>
            <label style={{ display: "flex", alignItems: "center", gap: "8px" }}>
              <input
                type="checkbox"
                checked={notifications}
                onChange={(e) => setNotifications(e.target.checked)}
              />
              Enable notifications
            </label>
            
            <label style={{ display: "flex", alignItems: "center", gap: "8px" }}>
              <input
                type="checkbox"
                checked={darkMode}
                onChange={(e) => setDarkMode(e.target.checked)}
              />
              Dark mode
            </label>
            
            <label style={{ display: "flex", alignItems: "center", gap: "8px" }}>
              <input
                type="checkbox"
                checked={autoSave}
                onChange={(e) => setAutoSave(e.target.checked)}
              />
              Auto-save
            </label>
          </div>
          
          <div style={{ marginTop: "16px", display: "flex", justifyContent: "flex-end" }}>
            <Popover.CloseTrigger asChild>
              <Button size="sm">Done</Button>
            </Popover.CloseTrigger>
          </div>
        </Popover.Content>
      </Popover.Positioner>
    </Popover.Root>
  )
}
```

## API Reference

### Popover.Root

The root container for the popover.

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `open` | `boolean` | `false` | Whether the popover is open |
| `onOpenChange` | `(open: boolean) => void` | - | Callback when open state changes |
| `modal` | `boolean` | `false` | Whether the popover is modal |
| `portalled` | `boolean` | `true` | Whether to render in a portal |

### Popover.Trigger

The element that triggers the popover. Must be used with `asChild`.

### Popover.Positioner

Controls the positioning of the popover content.

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `side` | `"top" \| "right" \| "bottom" \| "left"` | `"bottom"` | Preferred side to render |
| `align` | `"start" \| "center" \| "end"` | `"center"` | Alignment relative to trigger |
| `sideOffset` | `number` | `4` | Distance from trigger element |
| `alignOffset` | `number` | `0` | Additional alignment offset |

### Popover.Content

The main content area of the popover.

### Popover.CloseTrigger

A button that closes the popover when clicked.

### Popover.Arrow

An optional arrow pointing to the trigger element.

## Accessibility

The Popover component follows WAI-ARIA guidelines:

- Uses `role="dialog"` for the content
- Implements focus management and focus trap when modal
- Supports keyboard navigation and escape to close
- Properly links trigger and content with ARIA attributes
- Provides screen reader announcements
- Supports both mouse and keyboard interactions

## Best Practices

- Use for interactive content that requires user input
- Keep content focused and avoid overwhelming users
- Provide clear close actions and escape routes
- Consider mobile responsiveness and touch interactions
- Use appropriate positioning to avoid viewport clipping
- Implement proper focus management for accessibility
- Avoid nesting popovers as it creates poor UX
