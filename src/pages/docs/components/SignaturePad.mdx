# SignaturePad

An interactive signature capture component that allows users to draw signatures using mouse, touch, or stylus input. Perfect for contracts, forms, delivery confirmations, or any scenario requiring digital signature collection with export capabilities.

## Installation

```bash
npm install liquidify-react
```

## Import

```tsx
import { SignaturePad } from "liquidify-react/signaturePad";
```

## Usage

### Multi-Signature Document

```tsx
import { useState } from "react";
import { SignaturePad } from "liquidify-react/signaturePad";

export default function MultiSignatureDocument() {
  const [signatures, setSignatures] = useState({
    party1: null,
    party2: null,
    witness: null
  });

  const parties = [
    { key: "party1", name: "Party 1 (Seller)", required: true },
    { key: "party2", name: "Party 2 (Buyer)", required: true },
    { key: "witness", name: "Witness (Optional)", required: false }
  ];

  const handleSignature = (party, details) => {
    setSignatures(prev => ({ ...prev, [party]: details }));
  };

  const isComplete = () => {
    return signatures.party1 && signatures.party2;
  };

  const handleSubmit = () => {
    if (!isComplete()) {
      alert("Please ensure both parties have signed");
      return;
    }

    console.log("Document signed by all parties:", signatures);
    alert("Document successfully signed by all required parties!");
  };

  return (
    <div style={{ maxWidth: "800px", margin: "0 auto", padding: "2rem" }}>
      <h1>Legal Agreement Document</h1>

      <div style={{ backgroundColor: "#f8f9fa", padding: "2rem", borderRadius: "8px", marginBottom: "2rem" }}>
        <h2>Agreement Terms</h2>
        <p>This agreement is made between Party 1 and Party 2 for the following terms:</p>
        <ul>
          <li>Both parties agree to the terms and conditions outlined herein</li>
          <li>This agreement is legally binding and enforceable</li>
          <li>All signatures must be provided electronically</li>
          <li>This document serves as a legal record of the agreement</li>
        </ul>
      </div>

      <div style={{ display: "grid", gap: "2rem" }}>
        {parties.map(party => (
          <div key={party.key} style={{
            border: "1px solid #dee2e6",
            borderRadius: "8px",
            padding: "1.5rem",
            backgroundColor: "white"
          }}>
            <h3>{party.name} {party.required && <span style={{ color: "#dc3545" }}>*</span>}</h3>

            <SignaturePad.Root
              onDrawEnd={(details) => handleSignature(party.key, details)}
            >
              <SignaturePad.Control
                style={{
                  border: signatures[party.key] ? "2px solid #28a745" : "2px dashed #ccc",
                  borderRadius: "8px",
                  height: "150px",

                }}
              >
                <SignaturePad.Guide
                  style={{
                    color: signatures[party.key] ? "#28a745" : "#666",
                    fontSize: "14px",
                    textAlign: "center",
                    lineHeight: "150px"
                  }}
                >
                  {signatures[party.key] ? "✓ Signed" : "Sign here"}
                </SignaturePad.Guide>
              </SignaturePad.Control>

              <div style={{ marginTop: "1rem", display: "flex", gap: "1rem" }}>
                <SignaturePad.ClearTrigger
                  style={{
                    padding: "0.5rem 1rem",

                    border: "none",

                )}

          </div>
        ))}
      </div>

      <div style={{ marginTop: "2rem", textAlign: "center" }}>
        <button
          onClick={handleSubmit}
```

## API Reference

### SignaturePad.Root

The root container for the signature pad component.

#### Props

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `onDrawStart` | `(details: DrawDetails) => void` | - | Callback fired when drawing starts |
| `onDrawEnd` | `(details: DrawDetails) => void` | - | Callback fired when drawing ends |
| `onDraw` | `(details: DrawDetails) => void` | - | Callback fired during drawing |
| `disabled` | `boolean` | `false` | Whether the signature pad is disabled |
| `children` | `ReactNode` | - | The signature pad content |

### SignaturePad.Label

An optional label for the signature pad.

#### Props

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `children` | `ReactNode` | - | The label content |

### SignaturePad.Control

The interactive canvas area for drawing signatures.

#### Props

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `children` | `ReactNode` | - | The control content (usually includes Guide) |

### SignaturePad.Guide

A guide or placeholder text shown in the signature area.

#### Props

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `children` | `ReactNode` | - | The guide content |

### SignaturePad.ClearTrigger

A button to clear the signature.

#### Props

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `children` | `ReactNode` | - | The trigger content |

## Best Practices

1. **Clear Instructions**: Always provide clear guidance on where and how to sign
2. **Size Appropriately**: Ensure the signature area is large enough for comfortable signing
3. **Visual Feedback**: Provide clear visual feedback when signatures are captured
4. **Validation**: Validate that signatures are provided when required
5. **Export Options**: Consider providing options to export signatures as images
6. **Legal Compliance**: Ensure digital signatures meet legal requirements for your use case
7. **Privacy**: Handle signature data securely and in compliance with privacy regulations
8. **Cross-Device**: Test signature capture across different devices and input methods
