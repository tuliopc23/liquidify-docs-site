# Importing

## Import Strategies

LiqUIdify offers flexible import options to optimize your bundle size and development experience. Choose the strategy that best fits your project's needs.

## Import Methods

### CSS Styles (Required)

Import the global styles once in your app's entry point:

```tsx
// Import this once in your main app file
import "liquidify-react/styles";
```

**Where to import:**

- Next.js: `app/layout.tsx` or `pages/_app.tsx`
- Vite: `src/main.tsx`
- CRA: `src/index.tsx`
- Remix: `app/root.tsx`

### Component Imports

#### Subpath Imports (Recommended)

Import individual components from their dedicated paths for optimal tree-shaking. Base components are available at the top level, while interactive components built on Ark UI live under the `ark-ui` subpath:

```tsx
// Single component imports
import { Button } from "liquidify-react/button";
import { Card } from "liquidify-react/card";
// Ark UI components use the `ark-ui` subpath
import { Dialog } from "liquidify-react/ark-ui/dialog";
import { Checkbox } from "liquidify-react/ark-ui/checkbox";
```

**Benefits:**

- **Smaller bundles** - Only import what you use
- **Faster builds** - Less code to process
- **Better performance** - Reduced runtime overhead

#### Root Imports

Import multiple components from the main entry (useful when prototyping):

```tsx
// Multiple components from root
import { Button, Card, Dialog } from "liquidify-react";
```

**Use when:**

- Prototyping quickly
- Using many components
- Bundle size isn't critical

## Advanced Import Patterns

### Namespace Imports

Organize components under a namespace:

```tsx
import * as LiqUIdify from "liquidify-react/button";

function App() {
  return <LiqUIdify.Button>Click me</LiqUIdify.Button>;
}
```

### Grouped Imports

Import related components together:

```tsx
// Form components
import { Checkbox } from "liquidify-react/ark-ui/checkbox";
import { Select } from "liquidify-react/ark-ui/select";
import { Button } from "liquidify-react/button";

// Layout & navigation
import { Card } from "liquidify-react/card";
import { Tabs } from "liquidify-react/ark-ui/tabs";
```

### Conditional Imports

Load components conditionally using dynamic imports:

```tsx
import { lazy } from "react";

// Lazy load heavy components
const Dialog = lazy(() =>
  import("liquidify-react/ark-ui/dialog").then((m) => ({ default: m.Dialog })),
);
```

## Bundle Optimization

### Tree Shaking Setup

Ensure your bundler supports tree shaking:

#### Vite

```tsx
// vite.config.ts
export default defineConfig({
  build: {
    rollupOptions: {
      treeshake: true,
    },
  },
});
```

#### Next.js

```tsx
// next.config.js
module.exports = {
  experimental: {
    optimizePackageImports: ["liquidify-react"],
  },
};
```

#### Webpack

```tsx
// webpack.config.js
module.exports = {
  optimization: {
    usedExports: true,
    sideEffects: false,
  },
};
```

### Bundle Analysis

Check which components are included in your bundle:

```bash
# Analyze with webpack-bundle-analyzer
npm install -D webpack-bundle-analyzer
npx webpack-bundle-analyzer build/static/js/*.js

# Analyze with Next.js bundle analyzer
npm install -D @next/bundle-analyzer
ANALYZE=true npm run build
```

## Component Reference

### Available Subpaths

Most interactive components are under the `ark-ui` namespace. Here are common examples:

```tsx
// Base components
import { Button } from "liquidify-react/button";
import { Card } from "liquidify-react/card";
import { Badge } from "liquidify-react/badge";
import { IconButton } from "liquidify-react/iconButton";

// Ark UI components
import { Accordion } from "liquidify-react/ark-ui/accordion";
import { Tabs } from "liquidify-react/ark-ui/tabs";
import { Dialog } from "liquidify-react/ark-ui/dialog";
import { Checkbox } from "liquidify-react/ark-ui/checkbox";
import { RadioGroup } from "liquidify-react/ark-ui/radioGroup";
import { Select } from "liquidify-react/ark-ui/select";
import { Tooltip } from "liquidify-react/ark-ui/tooltip";
import { Menu } from "liquidify-react/ark-ui/menu";
```

## TypeScript Support

### Type Imports

Import types alongside components:

```tsx
import { Button, type ButtonProps } from "liquidify-react/button";
import { Card, type CardProps } from "liquidify-react/card";

interface MyComponentProps {
  buttonProps?: ButtonProps;
  cardProps?: CardProps;
}
```

### Generic Types

Use component generics for type safety:

```tsx
import { Select, type SelectProps } from "liquidify-react/ark-ui/select";

interface User {
  id: string;
  name: string;
}

const UserSelect: React.FC<SelectProps<User>> = (props) => {
  return <Select {...props} />;
};
```

## Best Practices

### Import Organization

Organize imports in this order:

```tsx
// 1. React and third-party libraries
import React, { useState } from "react";
import { motion } from "framer-motion";

// 2. Internal utilities and hooks
import { useLocalStorage } from "@/hooks/useLocalStorage";

// 3. LiqUIdify components (grouped by type)
import { Button } from "liquidify-react/button";
import { Card } from "liquidify-react/card";
import { Dialog } from "liquidify-react/ark-ui/dialog";

// 4. Local components
import { UserProfile } from "./UserProfile";
```

### Performance Tips

- **Use subpath imports** for production builds
- **Group related imports** to reduce import statements
- **Lazy load** components that aren't immediately needed
- **Analyze your bundle** regularly to catch unnecessary imports

### Development Experience

- **Use root imports** during development for speed
- **Switch to subpath imports** before production
- **Set up IDE autocomplete** for faster import discovery
- **Use TypeScript** for better import validation

## Common Patterns

### Component Library Wrapper

Create a wrapper module for your app:

```tsx
// src/components/ui.ts
export { Button } from "liquidify-react/button";
export { Card } from "liquidify-react/card";
export { Dialog } from "liquidify-react/ark-ui/dialog";
export { Checkbox } from "liquidify-react/ark-ui/checkbox";
export { Select } from "liquidify-react/ark-ui/select";
// ... other components

// Usage in your app
import { Button, Card, Dialog, Checkbox, Select } from "@/components/ui";
```

### Custom Hook Integration

Combine imports with custom hooks:

```tsx
import { Dialog } from "liquidify-react/ark-ui/dialog";
import { useDialog } from "@/hooks/useDialog";

function MyComponent() {
  const { isOpen, open, close } = useDialog();

  return (
    <Dialog.Root open={isOpen} onOpenChange={close}>
      {/* Dialog content */}
    </Dialog.Root>
  );
}
```

## Troubleshooting

### Import Errors

**Module not found:**

```bash
# Ensure the component exists (Ark UI components live under the ark-ui folder)
ls node_modules/liquidify-react/dist/libs/components/components/
ls node_modules/liquidify-react/dist/libs/components/components/ark-ui/
```

**Type errors:**

```tsx
// Make sure you're importing from the correct path
import { Button } from "liquidify-react/button"; // ✅ Correct
import { Button } from "liquidify-react/buttons"; // ❌ Wrong
```

**CSS not loading:**

```tsx
// Ensure styles are imported once in your app
import "liquidify-react/styles"; // ✅ Required
```
