# Design System

## Apple-Inspired Design for the Web

LiqUIdify's design system translates Apple's Human Interface Guidelines into web-native patterns, providing a cohesive visual language that feels familiar to Apple ecosystem users while embracing web platform capabilities.

## Core Design Principles

### Clarity
Every element serves a purpose and communicates clearly. Visual hierarchy guides users through interfaces without cognitive overhead.

```tsx
import { Card } from 'liquidify-react/card'
import { Badge } from 'liquidify-react/badge'

// Clear visual hierarchy with semantic meaning
<Card.Root>
  <Card.Header>
    <div className="flex items-center justify-between">
      <Card.Title>Project Update</Card.Title>
      <Badge variant="success">Complete</Badge>
    </div>
    <Card.Description>
      Clear, concise descriptions provide necessary context
    </Card.Description>
  </Card.Header>
</Card.Root>
```

### Consistency
Consistent patterns reduce cognitive load and build user confidence. Familiar interactions work predictably across all components.

```tsx
import { Button } from 'liquidify-react/button'
import { Dialog } from 'liquidify-react/dialog'

// Consistent action patterns across contexts
<div className="flex gap-3">
  <Button variant="default">Primary Action</Button>
  <Button variant="outline">Secondary Action</Button>
  <Button variant="ghost">Tertiary Action</Button>
</div>

// Same patterns in different contexts
<Dialog.Footer>
  <Button variant="outline">Cancel</Button>
  <Button variant="default">Confirm</Button>
</Dialog.Footer>
```

### Deference
Interface elements step back to highlight content. Chrome serves content, never competing with it.

```tsx
import { Navbar } from 'liquidify-react/navbar'
import { Card } from 'liquidify-react/card'

// Subtle navigation that doesn't compete with content
<Navbar.Root>
  <Navbar.Brand>App Name</Navbar.Brand>
  <Navbar.Content>
    {/* Minimal, focused navigation */}
  </Navbar.Content>
</Navbar.Root>

// Cards provide structure without overwhelming content
<Card.Root className="border-subtle">
  <Card.Content>
    {/* Content is the hero */}
  </Card.Content>
</Card.Root>
```

## Component Patterns

### Composition over Configuration

Components are designed for flexible composition:

```tsx
import { Card } from 'liquidify-react/card'
import { Avatar } from 'liquidify-react/avatar'
import { Badge } from 'liquidify-react/badge'

// Flexible composition patterns
<Card.Root>
  <Card.Header>
    <div className="flex items-center gap-3">
      <Avatar src="/user.jpg" fallback="JD" />
      <div className="flex-1">
        <Card.Title>John Doe</Card.Title>
        <Card.Description>Product Designer</Card.Description>
      </div>
      <Badge variant="success">Active</Badge>
    </div>
  </Card.Header>
</Card.Root>
```

### Progressive Disclosure

Reveal complexity gradually to avoid overwhelming users:

```tsx
import { Collapsible } from 'liquidify-react/collapsible'
import { Button } from 'liquidify-react/button'

// Progressive disclosure pattern
<Card.Root>
  <Card.Header>
    <Card.Title>Advanced Settings</Card.Title>
    <Card.Description>Basic configuration shown by default</Card.Description>
  </Card.Header>
  
  <Card.Content>
    {/* Essential options visible */}
    <div className="space-y-3">
      {/* Basic settings */}
    </div>
    
    <Collapsible.Root>
      <Collapsible.Trigger asChild>
        <Button variant="ghost" className="mt-4">
          Show advanced options
        </Button>
      </Collapsible.Trigger>
      <Collapsible.Content>
        {/* Advanced options revealed on demand */}
      </Collapsible.Content>
    </Collapsible.Root>
  </Card.Content>
</Card.Root>
```

## Best Practices

### Visual Hierarchy

Create clear information hierarchy:

1. **Start with content** - Design around your content, not the other way around
2. **Use contrast** - Differentiate elements through size, color, and spacing
3. **Guide the eye** - Use typography and spacing to create reading flow
4. **Be consistent** - Use the same patterns for similar content types

### Performance Considerations

Design choices that support performance:

- **System fonts** - No external font loading
- **CSS-based animations** - Hardware accelerated transitions
- **Minimal DOM** - Simple component structures
- **Tree-shakeable** - Only load what you use

### Accessibility Integration

Design with accessibility from the start:

- **Color contrast** - Meet WCAG standards automatically
- **Focus indicators** - Clear, visible focus states
- **Touch targets** - Minimum 44px tap areas
- **Readable fonts** - Optimized for clarity at all sizes

**Remember**: Great design feels invisible. LiqUIdify's design system enables you to create interfaces that feel natural and intuitive, allowing users to focus on their goals rather than learning your interface.
