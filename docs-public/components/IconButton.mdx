# IconButton

## Summary
A specialized button component designed specifically for icon-only interactions. Perfect for toolbar actions, navigation controls, and compact UI elements where space is limited.

### Import
```ts
import { IconButton } from 'liquidify-react/iconButton'
```

## Examples

### Basic Usage
```tsx
import { IconButton } from 'liquidify-react/iconButton'

function BasicIconButtons() {
  return (
    <div style={{ display: 'flex', gap: '8px' }}>
      <IconButton
        icon="search"
        aria-label="Search"
        onClick={() => console.log('Search clicked')}
      />
      <IconButton
        icon="plus"
        aria-label="Add new item"
        onClick={() => console.log('Add clicked')}
      />
      <IconButton
        icon="settings"
        aria-label="Open settings"
        onClick={() => console.log('Settings clicked')}
      />
    </div>
  )
}
```

### With Loading State
```tsx
import { IconButton } from 'liquidify-react/iconButton'

function LoadingIconButton() {
  const [isLoading, setIsLoading] = React.useState(false)

  const handleClick = async () => {
    setIsLoading(true)
    // Simulate async operation
    await new Promise(resolve => setTimeout(resolve, 2000))
    setIsLoading(false)
  }

  return (
    <IconButton
      icon="upload"
      loading={isLoading}
      aria-label="Upload file"
      onClick={handleClick}
    />
  )
}
```

### Toggle States
```tsx
import { IconButton } from 'liquidify-react/iconButton'
import { useState } from 'react'

function ToggleIconButton() {
  const [isFavorited, setIsFavorited] = useState(false)
  const [isBookmarked, setIsBookmarked] = useState(false)

  return (
    <div style={{ display: 'flex', gap: '8px' }}>
      <IconButton
        icon={isFavorited ? "heart" : "heart"}
        iconProps={{
          style: { color: isFavorited ? '#ff6b6b' : '#666' }
        }}
        variant="ghost"
        aria-label={isFavorited ? "Remove from favorites" : "Add to favorites"}
        onClick={() => setIsFavorited(!isFavorited)}
      />
      <IconButton
        icon={isBookmarked ? "bookmark" : "bookmark"}
        iconProps={{
          style: { color: isBookmarked ? '#2196f3' : '#666' }
        }}
        variant="ghost"
        aria-label={isBookmarked ? "Remove bookmark" : "Add bookmark"}
        onClick={() => setIsBookmarked(!isBookmarked)}
      />
    </div>
  )
}
```

## API Reference

### IconButton
A button component that displays only an icon.

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `icon` | `IconName` | - | **Required.** The icon to display |
| `iconProps` | `Omit<IconProps, "name">` | - | Props to customize the icon appearance |
| `variant` | `"primary" \| "secondary" \| "ghost" \| "danger" \| "success"` | `"primary"` | Button variant |
| `size` | `"xs" \| "sm" \| "md" \| "lg" \| "xl"` | `"md"` | Button size |
| `loading` | `boolean` | `false` | Shows loading spinner |
| `disabled` | `boolean` | `false` | Disables the button |
| `aria-label` | `string` | - | **Required for accessibility.** Describes the button's action |

### Available Icons
Common icons include: `search`, `plus`, `minus`, `edit`, `trash-2`, `settings`, `check`, `x`, `heart`, `star`, `home`, `menu`, `arrow-left`, `arrow-right`, `chevron-left`, `chevron-right`, `eye`, `eye-off`, `download`, `upload`, `play`, `pause`, `volume-2`, and many more.

## Accessibility

IconButton follows strict accessibility guidelines:

- **Always Required**: `aria-label` prop must be provided to describe the button's action
- **Keyboard Navigation**: Full keyboard support with Enter/Space activation
- **Screen Reader**: Proper ARIA labeling for assistive technologies
- **Focus Management**: Visible focus indicators and logical tab order

### ARIA Label Examples
```tsx
// ✅ Good - descriptive and actionable
<IconButton icon="trash-2" aria-label="Delete this item" />

// ✅ Good - context-specific
<IconButton icon="plus" aria-label="Add new team member" />

// ❌ Bad - too vague
<IconButton icon="settings" aria-label="Click here" />

// ❌ Bad - missing aria-label
<IconButton icon="search" />
```

## Best Practices

### Do's
- Always provide meaningful `aria-label` descriptions
- Use appropriate icon sizes that match your design system
- Consider loading states for async actions
- Group related icon buttons in toolbars or button groups
- Use consistent variants within the same context

### Don'ts
- Don't use icon buttons without text labels (except in established patterns like toolbars)
- Don't use generic icons that could be confused (e.g., multiple "settings" icons)
- Don't rely solely on color to convey meaning (use shape + context)
- Don't make touch targets too small on mobile (< 44px)

### Icon Selection Guidelines
- **Navigation**: `home`, `menu`, `chevron-left`, `chevron-right`
- **Actions**: `plus`, `edit`, `trash-2`, `settings`, `search`
- **Status**: `check`, `x`, `alert-circle`, `info`
- **Media**: `play`, `pause`, `volume-2`, `eye`, `eye-off`
- **Files**: `file`, `download`, `upload`

### Performance Tips
- Icons are optimized SVGs with minimal file size
- Lazy loading support for icon components
- CSS-in-JS styling for optimal performance
- Tree-shaking friendly imports
