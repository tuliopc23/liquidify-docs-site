# Steps

A multi-step process component that provides clear progress indication and navigation controls for guided workflows. Perfect for onboarding flows, checkout processes, form wizards, or any scenario requiring users to complete tasks in a specific sequence with visual progress tracking.

## Installation

```bash
npm install @liquidify/components
```

## Import

```tsx
import { Steps } from "@liquidify/components/steps";
```

## Usage

### Basic Checkout Flow

```tsx
import { useState } from "react";
import { Steps } from "@liquidify/components/steps";

export default function CheckoutSteps() {
  const [currentStep, setCurrentStep] = useState(0);

  const steps = [
    { title: "Cart", description: "Review items" },
    { title: "Shipping", description: "Delivery details" },
    { title: "Payment", description: "Payment method" },
    { title: "Confirmation", description: "Order complete" }
  ];

  return (
    <div style={{ maxWidth: "600px", margin: "0 auto" }}>
      <Steps.Root step={currentStep} count={steps.length}>
        <Steps.List>
          {steps.map((step, index) => (
            <Steps.Item key={index} index={index}>
              <Steps.Trigger onClick={() => setCurrentStep(index)}>
                <Steps.Indicator>{index + 1}</Steps.Indicator>
                <div>
                  <div style={{ fontWeight: "bold" }}>{step.title}</div>
                  <div style={{ fontSize: "0.9rem", color: "#666" }}>{step.description}</div>
                </div>
              </Steps.Trigger>
              {index < steps.length - 1 && <Steps.Separator />}
            </Steps.Item>
          ))}
        </Steps.List>

        <Steps.Content>
          <div style={{ padding: "2rem", textAlign: "center" }}>
            <h2>{steps[currentStep].title}</h2>
            <p>Content for step {currentStep + 1}</p>
          </div>
        </Steps.Content>

        <div style={{ display: "flex", justifyContent: "space-between", marginTop: "2rem" }}>
          <Steps.PrevTrigger
            onClick={() => setCurrentStep(Math.max(0, currentStep - 1))}
            disabled={currentStep === 0}
          >
            Previous
          </Steps.PrevTrigger>

          <Steps.NextTrigger
            onClick={() => setCurrentStep(Math.min(steps.length - 1, currentStep + 1))}
            disabled={currentStep === steps.length - 1}
          >
            Next
          </Steps.NextTrigger>
        </div>
      </Steps.Root>
    </div>
  );
}
```

### Form Wizard with Validation

```tsx
import { useState } from "react";
import { Steps } from "@liquidify/components/steps";

export default function FormWizard() {
  const [currentStep, setCurrentStep] = useState(0);
  const [formData, setFormData] = useState({
    personal: { name: "", email: "" },
    address: { street: "", city: "", zip: "" },
    payment: { cardNumber: "", expiry: "", cvv: "" }
  });
  const [errors, setErrors] = useState({});

  const steps = [
    {
      title: "Personal Info",
      description: "Your basic information",
      fields: ["name", "email"],
      validate: (data) => {
        const errs = {};
        if (!data.name) errs.name = "Name is required";
        if (!data.email) errs.email = "Email is required";
        else if (!/\S+@\S+\.\S+/.test(data.email)) errs.email = "Email is invalid";
        return errs;
      }
    },
    {
      title: "Address",
      description: "Shipping address",
      fields: ["street", "city", "zip"],
      validate: (data) => {
        const errs = {};
        if (!data.street) errs.street = "Street address is required";
        if (!data.city) errs.city = "City is required";
        if (!data.zip) errs.zip = "ZIP code is required";
        return errs;
      }
    },
    {
      title: "Payment",
      description: "Payment details",
      fields: ["cardNumber", "expiry", "cvv"],
      validate: (data) => {
        const errs = {};
        if (!data.cardNumber) errs.cardNumber = "Card number is required";
        if (!data.expiry) errs.expiry = "Expiry date is required";
        if (!data.cvv) errs.cvv = "CVV is required";
        return errs;
      }
    }
  ];

  const updateFormData = (section, field, value) => {
    setFormData(prev => ({
      ...prev,
      [section]: { ...prev[section], [field]: value }
    }));
    // Clear error when user starts typing
    if (errors[field]) {
      setErrors(prev => ({ ...prev, [field]: "" }));
    }
  };

  const validateCurrentStep = () => {
    const currentStepData = steps[currentStep];
    const validationErrors = currentStepData.validate(formData[currentStepData.title.toLowerCase().replace(" ", "")]);
    setErrors(validationErrors);
    return Object.keys(validationErrors).length === 0;
  };

  const handleNext = () => {
    if (validateCurrentStep()) {
      setCurrentStep(currentStep + 1);
    }
  };

  const renderFormFields = () => {
    const currentStepData = steps[currentStep];
    const section = currentStepData.title.toLowerCase().replace(" ", "");

    return (
      <div style={{ padding: "2rem" }}>
        <h3>{currentStepData.title}</h3>
        <form style={{ display: "flex", flexDirection: "column", gap: "1rem" }}>
          {currentStepData.fields.map(field => (
            <div key={field}>
              <label style={{ display: "block", marginBottom: "0.25rem" }}>
                {field.charAt(0).toUpperCase() + field.slice(1).replace(/([A-Z])/g, " $1")}
              </label>
              <input
                type={field === "email" ? "email" : "text"}
                value={formData[section][field]}
                onChange={(e) => updateFormData(section, field, e.target.value)}
                style={{
                  width: "100%",
                  padding: "0.5rem",
                  border: errors[field] ? "1px solid #dc3545" : "1px solid #ccc",
                  borderRadius: "4px"
                }}
              />
              {errors[field] && (
                <div style={{ color: "#dc3545", fontSize: "0.9rem", marginTop: "0.25rem" }}>
                  {errors[field]}
                </div>
              )}
            </div>
          ))}
        </form>
      </div>
    );
  };

  return (
    <div style={{ maxWidth: "600px", margin: "0 auto" }}>
      <Steps.Root step={currentStep} count={steps.length}>
        <Steps.List>
          {steps.map((step, index) => (
            <Steps.Item key={index} index={index}>
              <Steps.Trigger onClick={() => setCurrentStep(index)}>
                <Steps.Indicator>
                  {index < currentStep ? "✓" : index + 1}
                </Steps.Indicator>
                <div>
                  <div style={{ fontWeight: "bold" }}>{step.title}</div>
                  <div style={{ fontSize: "0.9rem", color: "#666" }}>{step.description}</div>
                </div>
              </Steps.Trigger>
              {index < steps.length - 1 && <Steps.Separator />}
            </Steps.Item>
          ))}
        </Steps.List>

        <Steps.Content>
          {renderFormFields()}
        </Steps.Content>

        <div style={{ display: "flex", justifyContent: "space-between", marginTop: "2rem" }}>
          <Steps.PrevTrigger
            onClick={() => setCurrentStep(Math.max(0, currentStep - 1))}
            disabled={currentStep === 0}
          >
            Previous
          </Steps.PrevTrigger>

          {currentStep < steps.length - 1 ? (
            <Steps.NextTrigger onClick={handleNext}>
              Next
            </Steps.NextTrigger>
          ) : (
            <button style={{ padding: "0.75rem 1.5rem", backgroundColor: "#28a745", color: "white", border: "none", borderRadius: "4px" }}>
              Submit Form
            </button>
          )}
        </div>
      </Steps.Root>
    </div>
  );
}
```

### Progress Indicator Only

```tsx
import { useState, useEffect } from "react";
import { Steps } from "@liquidify/components/steps";

export default function ProgressIndicator() {
  const [currentStep, setCurrentStep] = useState(0);
  const [isProcessing, setIsProcessing] = useState(false);

  const steps = [
    { title: "Uploading", description: "Sending files to server" },
    { title: "Processing", description: "Analyzing content" },
    { title: "Validating", description: "Checking for errors" },
    { title: "Complete", description: "All done!" }
  ];

  useEffect(() => {
    if (isProcessing && currentStep < steps.length - 1) {
      const timer = setTimeout(() => {
        setCurrentStep(prev => prev + 1);
      }, 2000); // Simulate 2 second processing time per step

      return () => clearTimeout(timer);
    } else if (currentStep === steps.length - 1) {
      setIsProcessing(false);
    }
  }, [currentStep, isProcessing]);

  const startProcess = () => {
    setCurrentStep(0);
    setIsProcessing(true);
  };

  return (
    <div style={{ maxWidth: "600px", margin: "0 auto", textAlign: "center" }}>
      <h2>File Processing</h2>

      <Steps.Root step={currentStep} count={steps.length}>
        <Steps.List>
          {steps.map((step, index) => (
            <Steps.Item key={index} index={index}>
              <Steps.Trigger>
                <Steps.Indicator>
                  {index < currentStep ? "✓" :
                   index === currentStep && isProcessing ? "⏳" :
                   index + 1}
                </Steps.Indicator>
                <div>
                  <div style={{
                    fontWeight: "bold",
                    color: index < currentStep ? "#28a745" :
                           index === currentStep ? "#007bff" : "#666"
                  }}>
                    {step.title}
                  </div>
                  <div style={{ fontSize: "0.9rem", color: "#666" }}>{step.description}</div>
                </div>
              </Steps.Trigger>
              {index < steps.length - 1 && <Steps.Separator />}
            </Steps.Item>
          ))}
        </Steps.List>
      </Steps.Root>

      <div style={{ marginTop: "2rem" }}>
        {!isProcessing && currentStep === 0 && (
          <button
            onClick={startProcess}
            style={{ padding: "0.75rem 1.5rem", backgroundColor: "#007bff", color: "white", border: "none", borderRadius: "4px" }}
          >
            Start Processing
          </button>
        )}

        {isProcessing && (
          <div>
            <p>Processing step {currentStep + 1} of {steps.length}...</p>
            <div style={{ width: "100%", height: "4px", backgroundColor: "#e9ecef", borderRadius: "2px", marginTop: "1rem" }}>
              <div
                style={{
                  width: `${((currentStep + 1) / steps.length) * 100}%`,
                  height: "100%",
                  backgroundColor: "#007bff",
                  borderRadius: "2px",
                  transition: "width 0.5s ease"
                }}
              />
            </div>
          </div>
        )}

        {!isProcessing && currentStep === steps.length - 1 && (
          <div style={{ color: "#28a745" }}>
            <h3>✅ Processing Complete!</h3>
            <p>All files have been successfully processed.</p>
          </div>
        )}
      </div>
    </div>
  );
}
```

## API Reference

### Steps.Root

The root container for the steps component.

#### Props

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `step` | `number` | `0` | Current active step index |
| `count` | `number` | - | Total number of steps |
| `orientation` | `"horizontal" \| "vertical"` | `"horizontal"` | Layout orientation |
| `children` | `ReactNode` | - | The steps content |

### Steps.List

Container for the step items.

#### Props

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `children` | `ReactNode` | - | The step items |

### Steps.Item

An individual step item.

#### Props

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `index` | `number` | - | The step index |
| `children` | `ReactNode` | - | The step item content |

### Steps.Trigger

The clickable trigger for a step.

#### Props

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `onClick` | `() => void` | - | Click handler |
| `children` | `ReactNode` | - | The trigger content |

### Steps.Indicator

The visual indicator for a step (number, checkmark, etc.).

#### Props

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `children` | `ReactNode` | - | The indicator content |

### Steps.Separator

The visual separator between steps.

#### Props

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `children` | `ReactNode` | - | The separator content |

### Steps.Content

Container for step-specific content.

#### Props

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `children` | `ReactNode` | - | The step content |

### Steps.NextTrigger

Button to navigate to the next step.

#### Props

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `onClick` | `() => void` | - | Click handler |
| `disabled` | `boolean` | `false` | Whether the trigger is disabled |
| `children` | `ReactNode` | - | The trigger content |

### Steps.PrevTrigger

Button to navigate to the previous step.

#### Props

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `onClick` | `() => void` | - | Click handler |
| `disabled` | `boolean` | `false` | Whether the trigger is disabled |
| `children` | `ReactNode` | - | The trigger content |

### Steps.Progress

Progress indicator component.

#### Props

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `children` | `ReactNode` | - | The progress content |

## Accessibility

The Steps component includes comprehensive accessibility features:

- **ARIA Roles**: Implements proper navigation and step roles
- **Keyboard Navigation**: Full keyboard support for step navigation
- **Focus Management**: Proper focus handling and visual indicators
- **Screen Reader Support**: Announces current step and navigation options
- **High Contrast**: Compatible with high contrast themes

### Keyboard Shortcuts

| Key | Action |
|-----|--------|
| `Tab` | Navigate between interactive elements |
| `Enter` / `Space` | Activate current step or trigger |
| `Arrow Left/Right` | Navigate between steps (horizontal) |
| `Arrow Up/Down` | Navigate between steps (vertical) |

## Best Practices

1. **Clear Progress Indication**: Always show current step and total steps
2. **Logical Step Order**: Ensure steps follow a logical sequence
3. **Validation**: Validate user input before allowing progression
4. **Save Progress**: Allow users to save progress and return later
5. **Mobile Friendly**: Ensure steps work well on mobile devices
6. **Error Handling**: Provide clear error messages and recovery options
7. **Skip Options**: Consider allowing users to skip optional steps
8. **Confirmation**: Provide confirmation before final submission
