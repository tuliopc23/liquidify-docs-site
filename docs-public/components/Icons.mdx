# Icons

## Summary
A comprehensive icon system built on Lucide React icons with consistent sizing and accessibility. Provides a unified API for displaying icons throughout your application with proper semantic markup.

### Import
```ts
import { Icon } from 'liquidify-react/icons'
```

## Examples

```tsx
import { Icon } from 'liquidify-react/icons'
```

## API Reference

### Icon
The main icon component that renders Lucide React icons.

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `name` | `IconName` | - | **Required.** The icon to display |
| `size` | `IconSize \| number` | `"md"` | Icon size (preset or custom number) |
| `className` | `string` | - | Additional CSS classes |
| `style` | `CSSProperties` | - | Inline styles |
| `title` | `string` | - | Tooltip/title text |

## Accessibility

The Icon component follows strict accessibility guidelines:

- **Screen Reader Support**: Icons are hidden from screen readers by default unless they have labels
- **ARIA Labels**: Use `aria-label` for interactive icons
- **Semantic Markup**: Proper SVG structure with accessibility attributes
- **Focus Management**: Icons inherit focus behavior from parent elements

### Making Icons Accessible
```tsx
// ✅ Accessible interactive icon
<Icon name="search" aria-label="Search" />

// ✅ Icon with tooltip
<Icon name="info" title="More information" />

// ✅ Decorative icon (hidden from screen readers)
<Icon name="chevron-right" />

// ❌ Missing label for interactive icon
<Icon name="settings" onClick={handleClick} />
```

## Best Practices

### Do's
- Always provide `aria-label` for interactive icons
- Use semantic icon names that match their purpose
- Choose appropriate sizes based on context
- Consider color contrast when styling icons
- Use consistent icon sizes within the same interface

### Don'ts
- Don't use icons without context or labels when meaning isn't clear
- Don't rely on color alone to convey meaning
- Don't make icons too small for touch interaction
- Don't overuse icons - combine with text when helpful
- Don't use generic icons for specific actions

### Icon Selection Guidelines
- **Navigation**: Use `home`, `chevron-left/right`, `menu`
- **Actions**: Use `plus`, `edit`, `trash-2`, `search`
- **Status**: Use `check-circle`, `alert-circle`, `x-circle`
- **Communication**: Use `mail`, `phone`, `message-circle`
- **User**: Use `user`, `users`, `log-in/out`

### Performance Tips
- Icons are tree-shaken - only used icons are bundled
- SVG-based for crisp rendering at any size
- No external dependencies for icon loading
- Optimized for fast rendering
