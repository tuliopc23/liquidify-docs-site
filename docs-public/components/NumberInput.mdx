# NumberInput

A specialized input component for numeric values with increment/decrement controls and validation.

## Installation

```bash
bun add @liquidify/components
```

## Import

```tsx
import { NumberInput } from "@liquidify/components/number-input"
```

## Basic Usage

Create a simple number input with increment/decrement buttons:

```tsx
import { NumberInput } from "@liquidify/components/number-input"

export function BasicExample() {
  return (
    <NumberInput.Root defaultValue="0">
      <NumberInput.Label>Quantity</NumberInput.Label>
      <NumberInput.Control>
        <NumberInput.DecrementTrigger>-</NumberInput.DecrementTrigger>
        <NumberInput.Input />
        <NumberInput.IncrementTrigger>+</NumberInput.IncrementTrigger>
      </NumberInput.Control>
    </NumberInput.Root>
  )
}
```

## Controlled Value

Control the number input value with state:

```tsx
import { useState } from "react"
import { NumberInput } from "@liquidify/components/number-input"

export function ControlledExample() {
  const [value, setValue] = useState("10")

  return (
    <NumberInput.Root value={value} onValueChange={(details) => setValue(details.value)}>
      <NumberInput.Label>Price ($)</NumberInput.Label>
      <NumberInput.Control>
        <NumberInput.DecrementTrigger>-</NumberInput.DecrementTrigger>
        <NumberInput.Input />
        <NumberInput.IncrementTrigger>+</NumberInput.IncrementTrigger>
      </NumberInput.Control>
      <p>Current value: ${value}</p>
    </NumberInput.Root>
  )
}
```

## Min/Max Values and Step

Set boundaries and increment steps:

```tsx
import { NumberInput } from "@liquidify/components/number-input"

export function MinMaxStepExample() {
  return (
    <NumberInput.Root 
      min={0} 
      max={100} 
      step={5} 
      defaultValue="50"
    >
      <NumberInput.Label>Progress (%)</NumberInput.Label>
      <NumberInput.Control>
        <NumberInput.DecrementTrigger>-</NumberInput.DecrementTrigger>
        <NumberInput.Input />
        <NumberInput.IncrementTrigger>+</NumberInput.IncrementTrigger>
      </NumberInput.Control>
      <NumberInput.HelperText>
        Value must be between 0 and 100, in steps of 5
      </NumberInput.HelperText>
    </NumberInput.Root>
  )
}
```

## Validation and Error States

Show validation errors and invalid states:

```tsx
import { useState } from "react"
import { NumberInput } from "@liquidify/components/number-input"

export function ValidationExample() {
  const [value, setValue] = useState("")
  const [error, setError] = useState("")

  const handleValueChange = (details) => {
    setValue(details.value)
    
    const numValue = parseFloat(details.value)
    if (details.value && (isNaN(numValue) || numValue < 1 || numValue > 10)) {
      setError("Value must be between 1 and 10")
    } else {
      setError("")
    }
  }

  return (
    <NumberInput.Root 
      value={value} 
      onValueChange={handleValueChange}
      min={1}
      max={10}
      invalid={!!error}
    >
      <NumberInput.Label>Rating (1-10)</NumberInput.Label>
      <NumberInput.Control>
        <NumberInput.DecrementTrigger>-</NumberInput.DecrementTrigger>
        <NumberInput.Input />
        <NumberInput.IncrementTrigger>+</NumberInput.IncrementTrigger>
      </NumberInput.Control>
      {error && (
        <NumberInput.ErrorText>{error}</NumberInput.ErrorText>
      )}
    </NumberInput.Root>
  )
}
```

## API Reference

### NumberInput.Root

The root container for the number input.

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `value` | `string` | - | Controlled value |
| `defaultValue` | `string` | `"0"` | Default value |
| `onValueChange` | `(details) => void` | - | Callback when value changes |
| `min` | `number` | - | Minimum allowed value |
| `max` | `number` | - | Maximum allowed value |
| `step` | `number` | `1` | Increment/decrement step |
| `precision` | `number` | - | Number of decimal places |
| `disabled` | `boolean` | `false` | Whether input is disabled |
| `invalid` | `boolean` | `false` | Whether input is invalid |
| `formatOptions` | `Intl.NumberFormatOptions` | - | Number formatting options |

### NumberInput.Label

The label for the number input.

### NumberInput.Control

Container for the input and increment/decrement buttons.

### NumberInput.Input

The main number input field.

### NumberInput.IncrementTrigger

Button to increment the value.

### NumberInput.DecrementTrigger

Button to decrement the value.

### NumberInput.HelperText

Helper text displayed below the input.

### NumberInput.ErrorText

Error message displayed when input is invalid.

## Accessibility

The NumberInput component follows WAI-ARIA guidelines:

- Uses `role="spinbutton"` for the input field
- Implements `aria-valuemin`, `aria-valuemax`, and `aria-valuenow`
- Supports keyboard navigation with arrow keys
- Provides screen reader announcements for value changes
- Associates labels and error messages with `aria-labelledby` and `aria-describedby`

## Best Practices

- Always provide clear labels indicating the expected value type
- Set appropriate min/max boundaries to prevent invalid inputs
- Use step values that make sense for your use case
- Provide helpful error messages for validation failures
- Consider using formatOptions for currency or percentage displays
- Test keyboard navigation and screen reader compatibility
