# Splitter

A resizable panel layout component that allows users to adjust the size of adjacent panels by dragging resize handles. Perfect for code editors, file explorers, comparison views, or any interface requiring flexible panel layouts with user-controlled sizing.

## Installation

```bash
npm install @liquidify/components
```

## Import

```tsx
import { Splitter } from "@liquidify/components/splitter";
```

## Usage

### Basic Horizontal Split

```tsx
import { Splitter } from "@liquidify/components/splitter";

export default function BasicHorizontalSplit() {
  return (
    <div style={{ height: "400px", border: "1px solid #ccc" }}>
      <Splitter.Root orientation="horizontal">
        <Splitter.Panel>
          <div style={{ padding: "1rem", height: "100%", backgroundColor: "#f8f9fa" }}>
            <h3>Left Panel</h3>
            <p>This is the left panel content. You can resize it by dragging the handle.</p>
            <ul>
              <li>Item 1</li>
              <li>Item 2</li>
              <li>Item 3</li>
            </ul>
          </div>
        </Splitter.Panel>

        <Splitter.ResizeTrigger />

        <Splitter.Panel>
          <div style={{ padding: "1rem", height: "100%", backgroundColor: "#e9ecef" }}>
            <h3>Right Panel</h3>
            <p>This is the right panel content. It adjusts automatically when you resize.</p>
            <div style={{ marginTop: "1rem" }}>
              <div style={{ width: "100%", height: "200px", backgroundColor: "#dee2e6", display: "flex", alignItems: "center", justifyContent: "center" }}>
                Content Area
              </div>
            </div>
          </div>
        </Splitter.Panel>
      </Splitter.Root>
    </div>
  );
}
```

## API Reference

### Splitter.Root

The root container for the splitter component.

#### Props

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `orientation` | `"horizontal" \| "vertical"` | `"horizontal"` | The orientation of the splitter |
| `size` | `number[]` | - | Controlled sizes of the panels |
| `onSizeChange` | `(sizes: number[]) => void` | - | Callback fired when panel sizes change |
| `children` | `ReactNode` | - | The splitter content |

### Splitter.Panel

An individual resizable panel within the splitter.

#### Props

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `defaultSize` | `number` | - | Default size of the panel (as percentage) |
| `minSize` | `number` | - | Minimum size of the panel (as percentage) |
| `maxSize` | `number` | - | Maximum size of the panel (as percentage) |
| `children` | `ReactNode` | - | The panel content |

### Splitter.ResizeTrigger

The draggable handle for resizing panels.

#### Props

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `disabled` | `boolean` | `false` | Whether the resize trigger is disabled |
| `children` | `ReactNode` | - | Custom trigger content |

## Accessibility

The Splitter component includes comprehensive accessibility features:

- **ARIA Roles**: Implements proper separator and adjustable roles
- **Keyboard Navigation**: Full keyboard support for resizing panels
- **Screen Reader Support**: Announces resize actions and panel information
- **Focus Management**: Proper focus handling for resize handles
- **High Contrast**: Compatible with high contrast themes

### Keyboard Shortcuts

| Key | Action |
|-----|--------|
| `Tab` | Move focus to resize handle |
| `Enter` / `Space` | Activate resize mode |
| `Arrow Keys` | Resize panels when in resize mode |
| `Escape` | Exit resize mode |

## Best Practices

1. **Size Constraints**: Always set appropriate min/max sizes to prevent panels from becoming unusable
2. **Visual Feedback**: Provide clear visual indicators for resize handles
3. **Default Sizes**: Set sensible default sizes based on content importance
4. **Responsive Design**: Consider how splitters behave on different screen sizes
5. **Performance**: Avoid putting heavy content in frequently resized panels
6. **Persistence**: Consider saving user panel sizes for better UX
7. **Touch Support**: Ensure resize handles are touch-friendly on mobile devices
8. **Loading States**: Handle loading states gracefully when panel content is dynamic
