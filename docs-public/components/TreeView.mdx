# TreeView

A hierarchical tree structure component that displays nested data with expandable and collapsible nodes. Perfect for file explorers, navigation menus, organizational charts, or any scenario requiring hierarchical data visualization with interactive expansion/collapse functionality.

## Installation

```bash
npm install @liquidify/components
```

## Import

```tsx
import { TreeView } from "@liquidify/components/treeView";
```

### Dynamic Tree with Data

```tsx
import { useState } from "react";
import { TreeView } from "@liquidify/components/treeView";

export default function DynamicTree() {
  const [selectedItem, setSelectedItem] = useState(null);

  const treeData = [
    {
      id: "root",
      name: "Project Root",
      type: "folder",
      children: [
        {
          id: "src",
          name: "src",
          type: "folder",
          children: [
            { id: "main", name: "main.ts", type: "file" },
            { id: "utils", name: "utils.ts", type: "file" },
            {
              id: "components",
              name: "components",
              type: "folder",
              children: [
                { id: "button", name: "Button.tsx", type: "file" },
                { id: "input", name: "Input.tsx", type: "file" }
              ]
            }
          ]
        },
        { id: "package", name: "package.json", type: "file" },
        { id: "readme", name: "README.md", type: "file" }
      ]
    }
  ];

  const renderTreeNode = (node) => {
    if (node.type === "file") {
      return (
        <TreeView.Item key={node.id}>
          <TreeView.ItemText
            style={{
              color: selectedItem === node.id ? "#007bff" : "inherit",
              cursor: "pointer"
            }}
            onClick={() => setSelectedItem(node.id)}
          >
            üìÑ {node.name}
          </TreeView.ItemText>
        </TreeView.Item>
      );
    }

    return (
      <TreeView.Branch key={node.id}>
        <TreeView.BranchControl>
          <TreeView.BranchIndicator>üìÅ</TreeView.BranchIndicator>
          <TreeView.BranchText>{node.name}</TreeView.BranchText>
        </TreeView.BranchControl>
        <TreeView.BranchContent>
          {node.children?.map(renderTreeNode)}
        </TreeView.BranchContent>
      </TreeView.Branch>
    );
  };

  return (
    <div style={{ width: "300px" }}>
      <TreeView.Root>
        <TreeView.Label>File Explorer</TreeView.Label>
        <TreeView.Tree>
          {treeData.map(renderTreeNode)}
        </TreeView.Tree>
      </TreeView.Root>

      {selectedItem && (
        <div style={{ marginTop: "1rem", padding: "1rem", border: "1px solid #ccc" }}>
          <h4>Selected File</h4>
          <p>Selected: {selectedItem}</p>
        </div>
      )}
    </div>
  );
}
```

### Tree with Search and Filter

```tsx
import { useState, useMemo } from "react";
import { TreeView } from "@liquidify/components/treeView";

export default function TreeWithSearch() {
  const [searchTerm, setSearchTerm] = useState("");

  const treeData = [
    {
      id: "fruits",
      name: "Fruits",
      children: [
        { id: "apple", name: "Apple" },
        { id: "banana", name: "Banana" },
        { id: "orange", name: "Orange" }
      ]
    },
    {
      id: "vegetables",
      name: "Vegetables",
      children: [
        { id: "carrot", name: "Carrot" },
        { id: "broccoli", name: "Broccoli" },
        { id: "spinach", name: "Spinach" }
      ]
    }
  ];

  const filteredData = useMemo(() => {
    if (!searchTerm) return treeData;

    const filterNode = (node) => {
      const matches = node.name.toLowerCase().includes(searchTerm.toLowerCase());
      if (matches) return node;

      if (node.children) {
        const filteredChildren = node.children
          .map(filterNode)
          .filter(Boolean);
        if (filteredChildren.length > 0) {
          return { ...node, children: filteredChildren };
        }
      }

      return null;
    };

    return treeData.map(filterNode).filter(Boolean);
  }, [searchTerm, treeData]);

  const renderTreeNode = (node) => {
    if (!node.children) {
      return (
        <TreeView.Item key={node.id}>
          <TreeView.ItemText>{node.name}</TreeView.ItemText>
        </TreeView.Item>
      );
    }

    return (
      <TreeView.Branch key={node.id}>
        <TreeView.BranchControl>
          <TreeView.BranchIndicator>üìÅ</TreeView.BranchIndicator>
          <TreeView.BranchText>{node.name}</TreeView.BranchText>
        </TreeView.BranchControl>
        <TreeView.BranchContent>
          {node.children.map(renderTreeNode)}
        </TreeView.BranchContent>
      </TreeView.Branch>
    );
  };

  return (
    <div style={{ width: "300px" }}>
      <div style={{ marginBottom: "1rem" }}>
        <input
          type="text"
          placeholder="Search..."
          value={searchTerm}
          onChange={(e) => setSearchTerm(e.target.value)}
          style={{ width: "100%", padding: "0.5rem" }}
        />
      </div>

      <TreeView.Root>
        <TreeView.Label>Food Categories</TreeView.Label>
        <TreeView.Tree>
          {filteredData.length > 0 ? (
            filteredData.map(renderTreeNode)
          ) : (
            <div style={{ padding: "1rem", color: "#666" }}>
              No items found
            </div>
          )}
        </TreeView.Tree>
      </TreeView.Root>
    </div>
  );
}
```

## API Reference

### TreeView.Root

The root container for the tree view component.

#### Props

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `children` | `ReactNode` | - | The tree view content |
| All standard div props | - | - | Standard HTML div attributes |

### TreeView.Label

An optional label for the tree view.

#### Props

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `children` | `ReactNode` | - | The label content |

### TreeView.Tree

The main tree container that holds all branches and items.

#### Props

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `children` | `ReactNode` | - | The tree content |

### TreeView.Branch

A branch node that can be expanded/collapsed to show child items.

#### Props

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `id` | `string` | - | Unique identifier for the branch |
| `children` | `ReactNode` | - | The branch content |

### TreeView.BranchControl

The clickable control area for a branch that handles expansion/collapse.

#### Props

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `children` | `ReactNode` | - | The control content |

### TreeView.BranchContent

The container for child items within a branch.

#### Props

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `children` | `ReactNode` | - | The branch content |

### TreeView.BranchIndicator

The visual indicator (usually an arrow or icon) showing branch state.

#### Props

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `children` | `ReactNode` | - | The indicator content |

### TreeView.BranchText

The text content for a branch node.

#### Props

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `children` | `ReactNode` | - | The branch text |

### TreeView.Item

A leaf node item that cannot be expanded.

#### Props

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `children` | `ReactNode` | - | The item content |

### TreeView.ItemIndicator

An optional indicator area for items (useful for checkboxes, icons, etc.).

#### Props

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `children` | `ReactNode` | - | The indicator content |

### TreeView.ItemText

The text content for an item node.

#### Props

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `children` | `ReactNode` | - | The item text |

## Accessibility

The TreeView component includes comprehensive accessibility features:

- **ARIA Roles**: Implements proper tree/treeitem roles for screen readers
- **Keyboard Navigation**: Full keyboard support for navigating through tree nodes
- **Focus Management**: Proper focus handling and visual indicators
- **Screen Reader Support**: Announces expansion state and navigation
- **High Contrast**: Compatible with high contrast themes

### Keyboard Shortcuts

| Key | Action |
|-----|--------|
| `Arrow Up/Down` | Navigate between visible items |
| `Arrow Left` | Collapse branch or move to parent |
| `Arrow Right` | Expand branch or move to first child |
| `Enter` / `Space` | Toggle branch expansion |
| `Home` | Move to first item |
| `End` | Move to last item |
| `*` | Expand all siblings at current level |

## Best Practices

1. **Clear Hierarchy**: Use consistent indentation and visual cues to show hierarchy levels
2. **Meaningful Icons**: Use appropriate icons to distinguish between different types of nodes
3. **Lazy Loading**: For large trees, consider lazy loading child nodes
4. **Search Functionality**: Provide search/filter capabilities for large datasets
5. **State Persistence**: Consider saving expansion state for user convenience
6. **Performance**: Virtualize large trees to maintain smooth performance
7. **Keyboard Navigation**: Ensure all functionality is accessible via keyboard
8. **Loading States**: Show loading indicators when fetching dynamic content
