# Switch

A toggle switch component for binary on/off states with visual feedback.

## Installation

```bash
bun add @liquidify/components
```

## Import

```tsx
import { Switch } from "@liquidify/components/switch"
```

## Basic Usage

Create a simple toggle switch:

```tsx
import { Switch } from "@liquidify/components/switch"

export function BasicExample() {
  return (
    <Switch.Root defaultChecked>
      <Switch.Control>
        <Switch.Thumb />
      </Switch.Control>
      <Switch.Label>Enable notifications</Switch.Label>
    </Switch.Root>
  )
}
```

## Controlled Switch

Control the switch state with your own state:

```tsx
import { useState } from "react"
import { Switch } from "@liquidify/components/switch"

export function ControlledExample() {
  const [isChecked, setIsChecked] = useState(false)

  return (
    <Switch.Root 
      checked={isChecked} 
      onCheckedChange={(details) => setIsChecked(details.checked)}
    >
      <Switch.Control>
        <Switch.Thumb />
      </Switch.Control>
      <Switch.Label>Dark mode</Switch.Label>
      <p>Dark mode is {isChecked ? "enabled" : "disabled"}</p>
    </Switch.Root>
  )
}
```

## Form Integration

Use switches in forms with validation:

```tsx
import { useState } from "react"
import { Switch } from "@liquidify/components/switch"
import { Button } from "@liquidify/components/button"

export function FormExample() {
  const [agreedToTerms, setAgreedToTerms] = useState(false)
  const [subscribeNewsletter, setSubscribeNewsletter] = useState(false)
  const [error, setError] = useState("")

  const handleSubmit = (e) => {
    e.preventDefault()
    
    if (!agreedToTerms) {
      setError("You must agree to the terms and conditions")
      return
    }
    
    setError("")
    console.log("Form submitted:", { agreedToTerms, subscribeNewsletter })
  }

  return (
    <form onSubmit={handleSubmit}>
      <div style={{ display: "flex", flexDirection: "column", gap: "16px" }}>
        <Switch.Root 
          checked={agreedToTerms}
          onCheckedChange={(details) => {
            setAgreedToTerms(details.checked)
            setError("")
          }}
        >
          <Switch.Control>
            <Switch.Thumb />
          </Switch.Control>
          <Switch.Label>
            I agree to the <a href="#" style={{ color: "#3b82f6" }}>terms and conditions</a>
          </Switch.Label>
        </Switch.Root>

        <Switch.Root 
          checked={subscribeNewsletter}
          onCheckedChange={(details) => setSubscribeNewsletter(details.checked)}
        >
          <Switch.Control>
            <Switch.Thumb />
          </Switch.Control>
          <Switch.Label>Subscribe to our newsletter</Switch.Label>
        </Switch.Root>

        {error && (
          <p style={{ color: "#ef4444", fontSize: "14px" }}>{error}</p>
        )}

        <Button type="submit" style={{ marginTop: "16px" }}>
          Create Account
        </Button>
      </div>
    </form>
  )
}
```

## API Reference

### Switch.Root

The root container for the switch component.

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `checked` | `boolean` | - | Controlled checked state |
| `defaultChecked` | `boolean` | `false` | Default checked state |
| `onCheckedChange` | `(details) => void` | - | Callback when checked state changes |
| `disabled` | `boolean` | `false` | Whether the switch is disabled |
| `invalid` | `boolean` | `false` | Whether the switch is invalid |
| `required` | `boolean` | `false` | Whether the switch is required |

### Switch.Control

The clickable control area containing the thumb.

### Switch.Thumb

The draggable thumb element that moves when toggled.

### Switch.Label

The label text for the switch component.

## Accessibility

The Switch component follows WAI-ARIA guidelines:

- Uses `role="switch"` for the control element
- Implements `aria-checked` to indicate the current state
- Supports keyboard navigation with Space and Enter keys
- Associates labels with `aria-labelledby`
- Provides proper focus management and visual indicators
- Supports screen reader announcements for state changes

## Best Practices

- Use clear, descriptive labels that explain what the switch controls
- Provide immediate visual feedback when the state changes
- Consider grouping related switches in settings panels
- Use consistent sizing across your application
- Ensure sufficient contrast for focus and disabled states
- Test keyboard navigation and screen reader compatibility
- Consider the impact of the toggle action before requiring confirmation
- Use switches for binary states, not for actions that require confirmation
