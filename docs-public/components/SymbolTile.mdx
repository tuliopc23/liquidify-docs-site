# SymbolTile

## Summary
A versatile symbol display component with multiple color tints. Perfect for displaying icons, symbols, or small visual elements with consistent styling and color theming.

### Import
```ts
import { SymbolTile } from 'liquidify-react/symbolTile'
```

## Examples

### Basic Usage
```tsx
import { SymbolTile } from 'liquidify-react/symbolTile'

function BasicSymbolTile() {
  return (
    <div style={{ display: 'flex', gap: '8px' }}>
      <SymbolTile>★</SymbolTile>
      <SymbolTile>♥</SymbolTile>
      <SymbolTile>♦</SymbolTile>
      <SymbolTile>♠</SymbolTile>
    </div>
  )
}
```

### File Type Icons
```tsx
import { SymbolTile } from 'liquidify-react/symbolTile'

const fileTypes = [
  { icon: '📄', name: 'Document', tint: 'blue' as const },
  { icon: '📊', name: 'Spreadsheet', tint: 'green' as const },
  { icon: '📈', name: 'Chart', tint: 'indigo' as const },
  { icon: '🎵', name: 'Audio', tint: 'teal' as const },
  { icon: '🎬', name: 'Video', tint: 'red' as const },
]

function FileTypeIcons() {
  return (
    <div style={{ display: 'flex', flexWrap: 'wrap', gap: '16px' }}>
      {fileTypes.map((file) => (
        <div key={file.name} style={{ textAlign: 'center' }}>
          <SymbolTile tint={file.tint} style={{ fontSize: '32px', marginBottom: '8px' }}>
            {file.icon}
          </SymbolTile>
          <div style={{ fontSize: '12px', color: '#666' }}>{file.name}</div>
        </div>
      ))}
    </div>
  )
}
```

## API Reference

### SymbolTile
A styled span element for displaying symbols and icons.

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `tint` | `"gray" \| "blue" \| "indigo" \| "teal"` | `"gray"` | Color tint for the symbol |
| `children` | `ReactNode` | - | Symbol or icon content (emoji, text, etc.) |
| `className` | `string` | - | Additional CSS classes |

### Supported Tints
- **gray**: Neutral gray color scheme
- **blue**: Blue color scheme
- **indigo**: Indigo/purple color scheme
- **teal**: Teal/green color scheme

## Accessibility

The SymbolTile component provides good accessibility support:

- **Semantic HTML**: Uses `<span>` element with appropriate semantics
- **Screen Reader**: Content is read naturally by assistive technologies
- **Keyboard Navigation**: Inherits keyboard behavior from parent context
- **Focus Management**: Can receive focus when used in interactive contexts

### ARIA Considerations

## Best Practices

### Do's
- Use appropriate tints that match your design system
- Keep symbols simple and recognizable
- Consider accessibility labels for interactive symbols
- Use consistent symbol sizes within the same context
- Combine with text labels when meaning isn't obvious

### Don'ts
- Don't use overly complex symbols or emojis
- Don't rely on color alone to convey meaning
- Don't make symbols too small for readability
- Don't overuse different tints in the same interface
- Don't use symbols without context when meaning is unclear

### Design Guidelines
- **Size**: Use consistent sizes (16px-32px) based on context
- **Spacing**: Maintain adequate spacing between symbol tiles
- **Contrast**: Ensure good contrast between symbol and background
- **Consistency**: Use the same symbols for the same actions throughout your app

### Performance Tips
- Use system fonts for best emoji rendering
- Consider using SVG icons for more precise control
- Lazy load symbol tiles when used in large lists
- Minimize re-renders by memoizing when appropriate
